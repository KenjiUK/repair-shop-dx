# 検索バー・タグスキャン UXレビュー（2025年ベストプラクティス）

## 現状の問題点

### 1. **検索バーの常時表示**
- TOPページで常に検索バーが表示されている
- 実際の使用頻度が不明確
- 画面スペースを占有

### 2. **タグスキャンボタンの重複**
- 検索バーでタグID検索が可能
- 別途タグスキャンボタンが存在（機能重複）
- モバイルで画面スペースを占有

### 3. **2025年UI/UXトレンドとの不一致**
- Progressive Disclosure（段階的開示）の原則に反する
- コンテキストに応じたアクションの不足
- ミニマルデザインの原則に反する

## 2025年最新UI/UXトレンド

### 1. **Progressive Disclosure（段階的開示）**
- 必要な時に必要な機能を表示
- 初期表示は最小限に
- ユーザーの意図に応じて展開

### 2. **Contextual Actions（コンテキストに応じたアクション）**
- 操作の文脈に応じた機能配置
- 頻繁に使う機能を優先表示
- 使用頻度の低い機能は折りたたみ

### 3. **Mobile-First Design**
- モバイルでの操作性を最優先
- 画面スペースの有効活用
- タッチターゲットの最適化

### 4. **Unified Search Experience**
- 検索とスキャンを統合
- 1つのインターフェースで複数の検索方法を提供
- 検索バー内にスキャン機能を統合

## 改善提案

### 提案1: 検索バーの折りたたみ化（推奨）

**実装方針:**
- 検索バーを折りたたみ可能にする
- 検索アイコンをクリック/タップで展開
- タグスキャン機能を検索バー内に統合

**メリット:**
- 画面スペースの節約
- 必要な時だけ表示（Progressive Disclosure）
- モバイルでの操作性向上

### 提案2: タグスキャンの統合

**実装方針:**
- 検索バー内にスキャンアイコンを配置
- 検索バー右側にスキャンボタンを統合
- 検索とスキャンを1つのインターフェースに

**メリット:**
- 機能の重複を解消
- UIの簡素化
- ユーザーの混乱を防止

### 提案3: フィルター機能の強化

**実装方針:**
- ステータスフィルターを強化
- 入庫区分フィルターを強化
- 検索の必要性を減らす

**メリット:**
- 検索バーの使用頻度を減らす
- より直感的な操作
- 画面スペースの節約

## 推奨実装

### オプションA: 折りたたみ式検索バー（推奨）

```
[検索アイコン] ← クリックで展開
  ↓ 展開時
[検索バー（タグスキャンアイコン統合）]
```

### オプションB: 検索バー削除 + フィルター強化

```
- 検索バーを削除
- ステータスフィルターと入庫区分フィルターで十分
- タグスキャンは別途ボタン（使用頻度が高い場合のみ）
```

### オプションC: 検索バー統合（現在の改善版）

```
[検索バー（タグスキャンアイコン内蔵）]
- 検索バー右側にスキャンアイコン
- タグスキャンボタンを削除
```

## 業務フロー考慮

### 検索バーの使用頻度
- **高頻度**: 顧客名・車名での検索（頻繁に使用）
- **中頻度**: ナンバーでの検索
- **低頻度**: タグIDでの検索（スキャンが主）

### タグスキャンの使用頻度
- **高頻度**: チェックイン時のタグ紐付け
- **中頻度**: タグIDでの検索（手入力よりスキャンが主）

## 最終推奨

**オプションA（折りたたみ式検索バー）を推奨**

理由:
1. 検索機能は必要だが、常時表示は不要
2. タグスキャンは検索バーに統合可能
3. 画面スペースを節約
4. 2025年のトレンドに合致
