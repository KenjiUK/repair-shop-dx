# サマリーカードをフィルターとして使用する提案

## 現状分析

### 現在の実装
1. **サマリーカード**
   - 本日の状況カード：各ステータスの件数表示
   - 入庫区分カード：各入庫区分の件数表示（既にクリック可能）
   - 長期プロジェクトカード：クリックで詳細表示

2. **フィルターエリア**
   - ステータスフィルター：水平スクロール可能
   - 入庫区分フィルター：未使用（`ServiceKindFilter`は存在するが使用されていない）

### 問題点
- **機能の重複**: サマリーカードとフィルターが別々に存在
- **画面スペースの無駄**: フィルターエリアが画面を占有
- **一貫性の欠如**: 同じ情報が2箇所に表示

## 提案：サマリーカードをフィルターとして使用

### メリット

#### 1. **UIの簡素化**
- フィルターエリアを削減または削除
- 画面スペースの節約
- 情報の一元化

#### 2. **一貫性の向上**
- サマリーカードとフィルターが統合
- ユーザーの混乱を防止
- 直感的な操作

#### 3. **2025年トレンドに合致**
- **Contextual Actions**: コンテキストに応じたアクション
- **Unified Interface**: 統合インターフェース
- **Progressive Disclosure**: 段階的開示

### 懸念点と対策

#### 1. **クリック可能であることの明確化**
- **対策**: ホバー効果、カーソルポインター、視覚的フィードバック
- **実装**: `cursor-pointer`, `hover:shadow-md`, `hover:scale-[1.02]`

#### 2. **アクティブフィルターの表示**
- **対策**: 選択中の項目を視覚的に強調
- **実装**: 背景色の変更、ボーダーの強調、アイコンの強調

#### 3. **フィルターリセット機能**
- **対策**: カードタイトルをクリックでリセット、または「すべて」項目を追加
- **実装**: カードタイトルにクリックハンドラを追加

## 実装方針

### オプションA: 完全統合（推奨）

**実装:**
- サマリーカードの各項目をクリック可能にする
- クリックでフィルターを適用
- フィルターエリアを削除
- アクティブフィルターを視覚的に強調

**メリット:**
- 最もシンプル
- 画面スペースの最大化
- 一貫性の向上

### オプションB: ハイブリッド

**実装:**
- サマリーカードをクリック可能にする
- フィルターエリアは残すが、簡略化
- 両方でフィルターを適用可能

**メリット:**
- 柔軟性の確保
- 段階的な移行が可能

### オプションC: フィルターエリアのみ

**実装:**
- 現在のフィルターエリアを維持
- サマリーカードは情報表示のみ

**メリット:**
- 変更が最小限
- 既存のUXを維持

## 推奨実装

**オプションA（完全統合）を推奨**

理由:
1. UIの簡素化と画面スペースの節約
2. 一貫性の向上
3. 2025年のトレンドに合致
4. ユーザーの混乱を防止（同じ情報が2箇所に表示されない）

## 実装の詳細

### 1. 本日の状況カード
- 各ステータス項目をクリック可能にする
- クリックでステータスフィルターを適用
- アクティブな項目を視覚的に強調

### 2. 入庫区分カード
- 既に`onServiceKindClick`が実装済み
- フィルター適用機能を追加
- アクティブな項目を視覚的に強調

### 3. フィルターエリア
- ステータスフィルターを削除
- 入庫区分フィルターは既に未使用

### 4. 視覚的フィードバック
- ホバー効果: `hover:shadow-md`, `hover:scale-[1.02]`
- アクティブ状態: 背景色の変更、ボーダーの強調
- カーソルポインター: `cursor-pointer`
