# フィルターエリア改善レビュー

## 📋 概要

フィルターエリアのUI/UXをレビューし、改善提案を行います。

---

## 🔍 確認された問題点

### 1. 並び替えセレクトの文字が見切れる

**問題:**
- `SelectTrigger` の幅が `w-[180px]` で固定されている
- 「入庫時間順（古い順）」などの長いテキストが表示しきれない

**現在の実装:**
```tsx
<SelectTrigger id="sort-option" className="w-[180px] h-12">
  <SelectValue />
</SelectTrigger>
```

**影響:**
- ユーザーが選択中の並び替えオプションを完全に確認できない
- 視認性が低下

---

### 2. ステータスフィルターのアイコン使用の不整合

**問題:**
- フィルターではステータスにアイコンを使用している
- JOBカードではステータスバッジにアイコンを使用していない
- 一貫性がない

**現在の実装:**
- フィルター: アイコン + テキスト
- JOBカード: テキストのみ（バッジ）

**影響:**
- ユーザーが混乱する可能性
- 視覚的な一貫性がない

---

### 3. セクションタイトルの視覚的区別が弱い

**問題:**
- 「ステータス」「入庫区分」「整備士」のセクションタイトルがテキストのみ
- 視覚的な区別が弱く、セクションの境界がわかりにくい

**現在の実装:**
```tsx
<Label className="text-base font-semibold text-slate-800 mb-2 block">ステータス</Label>
```

**影響:**
- セクションの区別がわかりにくい
- スクロール時にどのセクションか判断しにくい

---

### 4. 整備士の下のフィルター項目の意味が不明確

**問題:**
- 「緊急案件のみ」「重要顧客のみ」「部品調達待ち」「長期化部品調達」の意味が不明確
- 「部品調達待ち」がステータスフィルターと重複している

**現在の実装:**
```tsx
<Button>緊急案件のみ</Button>
<Button>重要顧客のみ</Button>
<Button>部品調達待ち</Button>  // ステータスフィルターと重複
<Button>長期化部品調達</Button>
```

**影響:**
- ユーザーがフィルターの意味を理解しにくい
- 重複により混乱する可能性

---

### 5. 「その他のメンテナンス」と「その他」のアイコンが同じ

**問題:**
- 両方とも `FileText` アイコンを使用
- 視覚的に区別できない

**現在の実装:**
```tsx
{ kind: "その他" as ServiceKind, icon: FileText, color: "text-slate-700" },
{ kind: "その他のメンテナンス" as ServiceKind, icon: FileText, color: "text-slate-700" },
```

**影響:**
- ユーザーが2つを区別しにくい
- クリック時に間違える可能性

---

## ✅ 改善提案

### 提案1: 並び替えセレクトの幅を調整

**修正内容:**
- `w-[180px]` → `w-auto min-w-[200px]` または `w-full sm:w-auto`
- テキストが完全に表示されるようにする

**実装例:**
```tsx
<SelectTrigger id="sort-option" className="w-full sm:w-auto min-w-[200px] h-12">
  <SelectValue />
</SelectTrigger>
```

---

### 提案2: ステータスフィルターのアイコンを削除（JOBカードと統一）

**修正内容:**
- ステータスフィルターからアイコンを削除
- JOBカードと同様にテキストのみで表示

**理由:**
- JOBカードでアイコンを使用していないため、統一性を保つ
- シンプルで分かりやすいUI

**実装例:**
```tsx
<Button>
  {label}  // アイコンを削除
  {count > 0 && <Badge>{count}</Badge>}
</Button>
```

---

### 提案3: セクションタイトルにアイコンを追加

**修正内容:**
- 各セクションタイトルにアイコンを追加
- 視覚的な区別を強化

**実装例:**
```tsx
<div className="flex items-center gap-2 mb-2">
  <Clock className="h-5 w-5 text-slate-600" />
  <Label className="text-base font-semibold text-slate-800">ステータス</Label>
</div>
```

**アイコン候補:**
- ステータス: `Clock` または `Activity`
- 入庫区分: `FileText` または `Package`
- 整備士: `Users`

---

### 提案4: 整備士の下のフィルター項目を整理

**修正内容:**
- 「部品調達待ち」を削除（ステータスフィルターと重複）
- セクションタイトルを追加して意味を明確化
- フィルター項目をグループ化

**実装例:**
```tsx
{/* 追加フィルター */}
<div className="mb-3">
  <Label className="text-base font-semibold text-slate-800 mb-2 block">
    追加フィルター
  </Label>
  <div className="flex flex-wrap gap-2">
    <Button>緊急案件のみ</Button>
    <Button>重要顧客のみ</Button>
    <Button>長期化部品調達</Button>
  </div>
</div>
```

---

### 提案5: 「その他のメンテナンス」のアイコンを変更

**修正内容:**
- 「その他のメンテナンス」のアイコンを `FileText` から別のアイコンに変更
- 視覚的に区別できるようにする

**アイコン候補:**
- `Wrench` (レンチ) - メンテナンス作業を表現
- `Settings` (設定) - その他の設定を表現
- `MoreHorizontal` (三点リーダー) - その他を表現

**推奨:**
- `Wrench` - メンテナンス作業であることが明確

**実装例:**
```tsx
{ kind: "その他のメンテナンス" as ServiceKind, icon: Wrench, color: "text-slate-700" },
```

**影響範囲:**
- `service-kind-summary-card.tsx` の `serviceKindConfig` を修正
- 他の関連コンポーネントも確認が必要

---

## 🎯 優先度

1. **高**: 並び替えセレクトの文字が見切れる問題（即座に修正）
2. **高**: 「その他のメンテナンス」のアイコン変更（視認性向上）
3. **中**: ステータスフィルターのアイコン削除（一貫性向上）
4. **中**: セクションタイトルにアイコン追加（視認性向上）
5. **中**: 整備士の下のフィルター項目の整理（明確化）
