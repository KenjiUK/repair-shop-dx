# UX改善: レイアウトと空欄問題の解決

## 問題点

### 1. 作業が増えると画面が拡大してしまう
- 作業カードが増えると、左側の車両画像セクションと右側の作業セクションが繋がって表示される
- これにより、画面が縦に長くなりすぎる
- 作業セクションと画像セクションを分離したデザインにすべき

### 2. 受付・引き渡しボタンがない場合の空欄問題
- 右上のアクションセクションが空欄になる
- これがベストプラクティスかどうか疑問

---

## 改善提案

### 提案A: 作業セクションを独立したスクロール可能エリアにする（推奨）

**コンセプト:**
- 左側の車両画像セクションは固定高さ
- 右側の作業セクションは独立したスクロール可能エリア
- 右上のアクションセクションがない場合は非表示

**UI構成:**
```
┌─────────────────────────────────────────┐
│ ジョブカード                            │
│ ┌──────────┐ ┌──────────────────────┐ │
│ │ 車両画像 │ │ 顧客情報・車両情報    │ │
│ │ (固定)   │ │ ステータス: 入庫済み  │ │
│ │          │ │                      │ │
│ │          │ │ [情報行]             │ │
│ │          │ │ [ボタン行]           │ │
│ │          │ │                      │ │
│ │          │ │ ┌──────────────────┐ │ │
│ │          │ │ │ 作業カード1      │ │ │
│ │          │ │ │ [診断を開始]     │ │ │
│ │          │ │ └──────────────────┘ │ │
│ │          │ │ ┌──────────────────┐ │ │
│ │          │ │ │ 作業カード2      │ │ │
│ │          │ │ │ [診断を開始]     │ │ │
│ │          │ │ └──────────────────┘ │ │
│ │          │ │ (スクロール可能)     │ │
│ └──────────┘ └──────────────────────┘ │
└─────────────────────────────────────────┘
```

**メリット:**
- 車両画像が常に見える
- 作業が増えても画面が拡大しない
- スクロールで全ての作業を確認可能

---

### 提案B: 右上の空欄を活用する

**コンセプト:**
- 右上のアクションセクションがない場合は、有用な情報を表示
- 例：進捗状況、次のアクション、重要情報など

**UI構成:**
```
┌─────────────────────────────────────────┐
│ ジョブカード                            │
│ ┌──────────┐ ┌──────────────────────┐ │
│ │ 車両画像 │ │ 顧客情報・車両情報    │ │
│ │          │ │ ステータス: 入庫済み  │ │
│ │          │ │                      │ │
│ │          │ │ [情報行]             │ │
│ │          │ │ [ボタン行]           │ │
│ │          │ │                      │ │
│ │          │ │ ┌──────────────────┐ │ │
│ │          │ │ │ 作業カード1      │ │ │
│ │          │ │ │ [診断を開始]     │ │ │
│ │          │ │ └──────────────────┘ │ │
│ │          │ └──────────────────────┘ │
│ │          │ ┌──────────────────────┐ │
│ │          │ │ 進捗状況              │ │
│ │          │ │ 作業1: 診断中         │ │
│ │          │ │ 作業2: 外注調整中     │ │
│ │          │ └──────────────────────┘ │
│ └──────────┘                           │
└─────────────────────────────────────────┘
```

**メリット:**
- 空欄を有効活用
- 追加情報を提供
- 画面の情報密度が向上

---

## 推奨案: 提案A + 提案Bの組み合わせ

### 実装方針

1. **作業セクションを独立したスクロール可能エリアにする**
   - 左側の車両画像セクションは固定高さ（例：300px）
   - 右側の作業セクションは`max-height`を設定し、スクロール可能にする

2. **右上のアクションセクションがない場合は非表示**
   - または、進捗状況や次のアクションを表示

3. **レスポンシブ対応**
   - モバイルでは縦に並べる
   - デスクトップでは横に並べる
