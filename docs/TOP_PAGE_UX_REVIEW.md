# TOPページ UXレビュー（業務目線）

## 📋 概要

TOPページ全体を業務目線でレビューし、仕様調整の提案を行います。

**レビュー日**: 2024年12月  
**対象**: トップページ全体（サマリーカード、フィルター、ジョブカード一覧）

---

## 🔍 現在の実装の分析

### 現在の構造

1. **サマリーカードセクション（上部）**
   - **優先度1（横並び）**:
     - 本日の作業状況（ステータス別集計、クリックでフィルター適用）
     - 入庫区分別（サービス種別集計、クリックでフィルター適用）
   - **SummaryCarousel（横スクロール）**:
     - 本日入出庫予定
     - 整備士別（クリックでフィルター適用）
     - 代車在庫
     - 長期プロジェクト
     - 業務分析

2. **フィルターセクション（中部）**
   - 検索バー
   - アクティブフィルター表示（バッジ形式）
   - フィルターオプション:
     - 緊急案件・重要顧客フィルター
     - 部品調達待ちフィルター
     - 長期化している承認待ち案件フィルター
   - 並び替えオプション

3. **ジョブカード一覧（下部）**
   - フィルター適用後のジョブカードを表示

### 業務フローの分析

**受付スタッフの典型的な業務フロー:**
1. **朝の準備（最重要）**: 本日の入出庫予定を確認
   - 入庫予定の確認（受付準備）
   - 出庫予定の確認（顧客連絡準備）
2. **受付業務**: 入庫待ちの案件を確認・処理
   - ステータス別の件数確認
   - 入庫区分別の件数確認
3. **状況把握**: 各ステータスの件数を確認
   - 一目で全体像を把握したい
4. **フィルター操作**: 特定の条件で案件を絞り込み
   - 整備士別、入庫区分別、ステータス別など
5. **案件処理**: ジョブカードをクリックして詳細処理

## 🎯 問題点の分析

### 1. 情報の優先順位の問題 ⚠️ 最重要

**現在の構造:**
- 本日の作業状況・入庫区分別が最上部（横並び）
- 本日入出庫予定がSummaryCarousel内（横スクロールが必要）

**問題:**
- **本日入出庫予定は業務上最重要だが、すぐに見えない**
  - 朝の準備時に最初に確認したい情報
  - 横スクロールが必要で、見落としやすい
  - 受付スタッフが最も必要とする情報が後回しになっている
- **入庫予定・出庫予定の情報が埋もれている**
  - 遅延案件の早期発見が困難

### 2. フィルター機能の重複と一貫性の問題

**現在の実装:**
- サマリーカード（本日の作業状況、入庫区分別、整備士別）でフィルター機能（クリックでフィルター適用）
- 下部のフィルターセクションでも同様の機能

**問題:**
- **同じ機能が2箇所にある（重複）**
  - 本日の作業状況 → ステータスフィルター
  - 入庫区分別 → サービス種別フィルター
  - 整備士別 → 整備士フィルター
- **ユーザーが混乱する可能性**
  - どこでフィルターを設定すべきか分からない
- **メンテナンスが複雑**
  - フィルターロジックが分散している

**しかし、現在の形式の利点:**
- ✅ **一目で状況を把握できる**（サマリーカードの利点）
  - 件数が視覚的に分かりやすい
  - クリックで即座にフィルター適用できる
  - 40歳以上ユーザーにとって分かりやすい

### 3. 画面スペースの効率性

**現在の構造:**
- サマリーカードが多く、画面スペースを多く使用
- ジョブカード一覧が下にスクロールしないと見えない

**問題:**
- 重要な情報（ジョブカード）が最初に見えない
- サマリーカードが多すぎて、実際の作業に必要な情報が後回し

### 4. フィルターセクションの使いやすさ

**現在の実装:**
- 検索バー、フィルターオプション、並び替えが縦に並んでいる
- アクティブフィルターがバッジ形式で表示される

**問題:**
- フィルターオプションが散在している
- 入庫区分別のフィルターが下部にない（サマリーカードのみ）

## 💡 改善提案

### 提案1: 本日入出庫予定を左上に配置（推奨）⭐

**実装方法:**
- 本日入出庫予定を最上部の左側に配置
- 本日の作業状況を右側に配置（現状維持、クリックでフィルター適用）
- 入庫区分別は下部のフィルターセクションにも追加（重複を許容）

**メリット:**
- ✅ 業務上最重要情報（入出庫予定）がすぐに見える
- ✅ 朝の準備時に必要な情報が最初に表示される
- ✅ 本日の作業状況は現状維持（一目で状況把握可能）

**デメリット:**
- ⚠️ レイアウトの変更が必要
- ⚠️ フィルター機能の重複が残る

### 提案2: 入庫区分別をフィルターに統合（完全統合）

**実装方法:**
- 入庫区分別カードを削除
- 下部のフィルターセクションに「入庫区分」フィルターを追加
- 本日の作業状況もフィルターに統合（オプション）

**メリット:**
- ✅ フィルター機能の重複を完全に解消
- ✅ 画面スペースを節約
- ✅ 一貫性のあるUI/UX

**デメリット:**
- ⚠️ サマリーカードの利便性が失われる可能性
- ⚠️ 一目で状況を把握しにくくなる可能性
- ⚠️ 40歳以上ユーザーにとって分かりにくくなる可能性

### 提案3: ハイブリッド形式（推奨）⭐

**実装方法:**
- 本日入出庫予定を左上に配置
- 本日の作業状況を右上に配置（現状維持、クリックでフィルター適用）
- 入庫区分別はサマリーカードとして維持（クリックでフィルター適用）
- **下部のフィルターセクションには「入庫区分」フィルターを追加しない**（役割分担を明確化）
- SummaryCarouselは整備士別、代車在庫、長期プロジェクト、業務分析のみ

**メリット:**
- ✅ 業務上最重要情報がすぐに見える
- ✅ サマリーカードの利便性を維持（一目で状況把握可能）
- ✅ 役割分担が明確（サマリーカード = ワンクリック、下部フィルター = 詳細操作）
- ✅ 画面スペースを効率的に使用
- ✅ 一貫性のあるUI/UX（重複を避ける）
- ✅ ユーザーの混乱を防ぐ（「どこで設定すべきか」が明確）

**デメリット:**
- ⚠️ 入庫区分のフィルター操作はサマリーカードのみ（下部フィルターでは操作不可）

### 提案4: 完全分離形式

**実装方法:**
- 本日入出庫予定を左上に配置
- 本日の作業状況・入庫区分別を削除
- すべてのフィルターを下部のフィルターセクションに統合
- SummaryCarouselは整備士別、代車在庫、長期プロジェクト、業務分析のみ

**メリット:**
- ✅ フィルター機能の重複を完全に解消
- ✅ 画面スペースを最大限に節約
- ✅ 一貫性のあるUI/UX

**デメリット:**
- ⚠️ サマリーカードの利便性が完全に失われる
- ⚠️ 一目で状況を把握しにくくなる
- ⚠️ 40歳以上ユーザーにとって分かりにくくなる可能性が高い

## 💡 詳細な実装提案

### 推奨構造（提案3）

```
┌─────────────────────────────────────────┐
│ 本日入出庫予定  │  本日の作業状況        │
│ [入庫予定 3件]  │  [入庫待ち 5件]       │
│ [出庫予定 2件]  │  [診断待ち 2件]       │
│                │  [作業待ち 1件]       │
└─────────────────────────────────────────┘
│ 入庫区分別      │  （将来の拡張用）      │
│ [車検 3件]      │                      │
│ [点検 2件]      │                      │
└─────────────────────────────────────────┘
│ SummaryCarousel（横スクロール）         │
│ [整備士別] [代車在庫] [長期プロジェクト] │
└─────────────────────────────────────────┘
│ 検索バー                                 │
│ [緊急案件] [重要顧客] [入庫区分▼] [並び替え▼] │
└─────────────────────────────────────────┘
│ ジョブカード一覧                         │
│ [JobCard] [JobCard] [JobCard] ...      │
└─────────────────────────────────────────┘
```

### 重要なポイント

1. **本日入出庫予定**: 左上に配置、業務上最重要
2. **本日の作業状況**: 右上に配置、現状維持（クリックでフィルター適用）
3. **入庫区分別**: 第2段階に配置、現状維持（クリックでフィルター適用）
4. **SummaryCarousel**: 補助的な情報のみ（整備士別、代車在庫、長期プロジェクト、業務分析）
5. **下部フィルター**: **入庫区分フィルターは追加しない**（サマリーカードで完結）

### 業務フローとの整合性

**朝の準備時:**
1. 本日入出庫予定を確認（左上、すぐに見える）
2. 本日の作業状況を確認（右上、一目で把握）

**受付業務時:**
1. 入庫区分別を確認（第2段階、一目で把握）
2. サマリーカードをクリックして即座にフィルター適用（直感的）
3. 必要に応じて下部フィルターで追加の絞り込み（緊急案件、重要顧客など）

**日常業務時:**
1. SummaryCarouselで補助情報を確認（整備士別、代車在庫など）
2. サマリーカード（整備士別）をクリックしてフィルター適用
3. 下部フィルターで詳細な絞り込み（複数条件の組み合わせ）

### 設計思想の再検討

**問題点:**
- サマリーカードと下部フィルターの重複は、ユーザーの混乱を招く可能性がある
- 「どこでフィルターを設定すべきか」が不明確
- メンテナンスが複雑になる

**最適解: 役割分担を明確化**

1. **サマリーカード**: 
   - **役割**: 情報表示 + ワンクリックフィルター適用（直感的操作）
   - **対象**: 本日の作業状況、入庫区分別、整備士別
   - **動作**: クリックで即座にフィルター適用、ジョブカード一覧にスクロール

2. **下部フィルター**:
   - **役割**: 詳細なフィルター操作、複数条件の組み合わせ
   - **対象**: 緊急案件、重要顧客、部品調達待ち、長期化承認待ち、検索バー、並び替え
   - **動作**: 複数のフィルターを組み合わせて詳細に絞り込み

**結論:**
- **入庫区分フィルターは下部フィルターに追加しない**
- サマリーカード（入庫区分別）で完結させる
- これにより、役割分担が明確になり、ユーザーの混乱を防ぐ
