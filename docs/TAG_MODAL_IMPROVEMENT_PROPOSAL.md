# タグ紐付けモーダルの改善提案

## 現在の実装

使用中のタグに「使用中」というラベルのみが表示されています。

## 問題点

- 「使用中」だけでは、誰が何を使っているか分からない
- 受付スタッフがタグを選択する際に、どのタグが空いているか判断しにくい

## 提案

### オプション1: タグボタンの下に顧客名・車両名を表示（推奨）

**メリット:**
- 一目で誰が何を使っているか分かる
- 情報が常に見える

**デメリット:**
- 長い名前の場合、表示が詰まって見える可能性

**実装:**
- タグボタンの下に小さなテキストで表示
- 長すぎる場合は省略（`truncate`）
- ツールチップで完全な情報を表示

### オプション2: ツールチップ（ホバー時に詳細表示）

**メリット:**
- 画面スペースを節約できる
- 必要な時だけ情報を表示

**デメリット:**
- ホバーしないと情報が見えない
- タッチデバイスでは使いにくい

### オプション3: バッジを拡張（顧客名のみ）

**メリット:**
- 既存のUIを活用できる
- シンプル

**デメリット:**
- 車両名が分からない
- バッジが長くなりすぎる可能性

## 推奨実装

**オプション1を推奨します。**

理由:
1. 情報が常に見えるため、受付スタッフが素早く判断できる
2. タッチデバイスでも使いやすい
3. 長い名前は省略表示し、ツールチップで完全な情報を表示できる

### 表示例

```
┌─────────┐
│   05    │ ← タグID（大きく表示）
│ 使用中  │ ← バッジ（右上）
└─────────┘
田中 太郎  ← 顧客名（小さく表示、省略可）
プリウス    ← 車両名（小さく表示、省略可）
```

### 実装詳細

1. `tag.jobId` を使って、`jobs` から該当するジョブを検索
2. ジョブから顧客名（`field4.name`）と車両名（`field6.name`）を取得
3. タグボタンの下に小さなテキストで表示
4. 長すぎる場合は `truncate` で省略
5. `title` 属性でツールチップを表示
