# UIUX改善実装チェックリスト

## 📋 実装前の確認事項（毎回必ず確認）

### 1. ボタンサイズの統一確認

#### ✅ チェック項目
- [ ] `button.tsx`の基本サイズが正しいか確認
  - `default`: `h-12` (48px)
  - `sm`: `h-10` (40px) ※デスクトップのみ、最小限の使用
  - `lg`: `h-14` (56px) ※重要アクション用
  - `icon`: `size-12` (48px)
  - `icon-sm`: `size-10` (40px)
  - `icon-lg`: `size-14` (56px)

- [ ] **カスタムサイズ指定がないか確認（重要）**
  - コンポーネント内で`className`に`h-7`, `h-8`, `h-9`, `h-10`, `h-11`, `h-12`, `h-14`, `h-16`などが直接指定されていないか
  - `Button`コンポーネントを使っている場合、`size`プロパティを使っているか確認
  - 同じような重要度のボタンが同じサイズになっているか確認

#### 🔍 検索方法
```bash
# カスタムサイズ指定を検索
grep -r "h-[0-9]" src/components/features/
grep -r "h-[0-9]" src/app/

# ボタン関連のカスタムサイズを検索
grep -r "className.*h-[0-9]" src/components/features/
grep -r "Button.*className.*h-[0-9]" src/
```

#### ⚠️ 注意点
- **同じような重要なアクションボタンは同じサイズになるべき**
  - 例：「受付を開始」と「引渡しを開始」は同じサイズ（`h-12`）
  - 例：同じページ内のプライマリアクションは同じサイズ
- `size`プロパティを使わず、`className`で直接サイズを指定している場合がある
- `priority === "high"`と`priority === "medium"`でも、同じような重要なアクションなら同じサイズに統一

## 🎯 よくある問題と対処法

### 問題1: ボタンサイズが統一されていない
**原因**: `Button`コンポーネントの`size`プロパティを使わず、`className`で直接サイズを指定している

**対処法**: 
1. `className`内の`h-*`を削除
2. `size`プロパティを使用
3. カスタムサイズが必要な場合は、`button.tsx`のサイズ定義を確認

### 問題2: 同じような重要なアクションが異なるサイズ
**原因**: `priority`やコンポーネントの種類によって異なるサイズが使われている

**対処法**: 
1. 同じような重要度のアクションは同じサイズに統一
2. `priority === "high"`と`priority === "medium"`でも、重要なアクションなら同じサイズを検討

### 問題3: 小さなフォントサイズが使われている
**原因**: 従来のデザインで`text-xs`や`text-sm`が使われている

**対処法**: 
1. `text-xs`は使用禁止（`text-base`に変更）
2. `text-sm`は最小限の使用（重要情報は`text-base`に変更）
