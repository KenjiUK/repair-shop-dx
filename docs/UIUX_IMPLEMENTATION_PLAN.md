# UIUX改善実装計画

## 📋 実装方針

1. **段階的実装**: Phase 1の最優先項目から順番に実装
2. **リバース可能**: 各変更はgitで管理し、必要に応じて元に戻せるようにする
3. **小さな単位で実装**: 大きな変更は分割して実装し、確認しながら進める
4. **検証**: 各変更後、`npm run lint`で確認

## 🔄 リバース方法

### 方法1: Gitを使ったリバース（推奨）

```bash
# 特定のファイルを元に戻す
git checkout HEAD -- src/components/ui/button.tsx

# すべての変更を元に戻す（注意：未保存の変更も失われます）
git restore .
```

### 方法2: 手動で元に戻す

各変更時に、変更前のコードをコメントで残すか、変更内容を明確に記載します。

## 📝 実装順序

### Phase 1.1: ボタンサイズの拡大と統一 ⭐ **最優先**

**変更内容:**
- `src/components/ui/button.tsx` のsize定義を変更
  - `default`: `h-9` (36px) → `h-12` (48px)
  - `sm`: `h-8` (32px) → `h-10` (40px) ※デスクトップのみ、最小限の使用
  - `lg`: `h-10` (40px) → `h-14` (56px) ※重要アクション用
  - `icon`: `size-9` (36px) → `size-12` (48px)
  - `icon-sm`: `size-8` (32px) → `size-10` (40px)
  - `icon-lg`: `size-10` (40px) → `size-14` (56px)
- ボタン内テキストサイズ: `text-sm` → `text-base` (16px)

**影響範囲:**
- 全ボタンコンポーネント（約30箇所）

**確認事項:**
- レイアウトの崩れがないか
- モバイルでの表示が適切か
- タッチターゲットサイズが48px以上になっているか

**リバース方法:**
```bash
git checkout HEAD -- src/components/ui/button.tsx
```

---

### Phase 1.2: タイポグラフィの拡大 ⭐ **最優先**

**変更内容:**
- `text-xs` (12px) の使用を禁止
- `text-sm` (14px) → `text-base` (16px) に変更（ラベル・補足情報）
- コントラスト向上: `text-slate-500`, `text-slate-600` → `text-slate-700`, `text-slate-800`

**影響範囲:**
- 約150箇所のファイル

**実装アプローチ:**
1. よく使われるコンポーネントから順に修正
2. グローバルスタイル（`globals.css`）の確認
3. 主要ページ（`page.tsx`, `job-card.tsx`など）から順に修正

**確認事項:**
- テキストが読みやすくなっているか
- レイアウトの崩れがないか
- コントラスト比が4.5:1以上になっているか

**リバース方法:**
各ファイルごとに元に戻す必要があります。変更前に`git diff`で確認できます。

---

### Phase 1.3: カラーパレットの統一とコントラスト向上 ⭐ **最優先**

**変更内容:**
- ステータスカラーのテキスト: `-800` → `-900`（コントラスト向上）
- ボーダーカラー: `-200` → `-300`（視認性向上）
- `text-slate-500`, `text-slate-600` の使用を最小限に

**影響範囲:**
- 約50箇所

**確認事項:**
- コントラスト比が4.5:1以上になっているか
- 色の識別性が向上しているか

---

### Phase 1.4: アイコンサイズの拡大

**変更内容:**
- ボタン内アイコン: `size-4` → `size-5` (20px)
- カードタイトルアイコン: `size-5` → `size-6` (24px)
- バッジ内アイコン: `size-3` → `size-4` (16px)
- `size-3` (12px) の使用を禁止

**影響範囲:**
- 約150箇所

---

### Phase 2以降

Phase 1完了後、Phase 2以降の実装計画を再確認します。

## 🔍 変更前の確認方法

```bash
# 変更内容を確認
git diff src/components/ui/button.tsx

# 変更されたファイル一覧を確認
git status

# 特定のコミットまで戻す（コミットハッシュを指定）
git checkout <commit-hash> -- src/components/ui/button.tsx
```

## ✅ 各実装後の確認チェックリスト

- [ ] `npm run lint` でエラーがないか確認
- [ ] `npm run dev` で動作確認
- [ ] ブラウザで実際の表示を確認
- [ ] モバイル表示を確認（必要に応じて）
- [ ] タッチターゲットサイズが適切か確認

## 📌 注意事項

1. **大きな変更は事前に確認**: レイアウトに大きな影響を与える変更は、実装前に確認します
2. **段階的に実装**: 一度にすべてを変更せず、小さな単位で実装・確認を繰り返します
3. **バックアップ**: 重要な変更前は、git commitで状態を保存します（必要に応じて）
