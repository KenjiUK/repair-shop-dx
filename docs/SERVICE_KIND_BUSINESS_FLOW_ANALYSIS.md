# 入庫区分別業務フロー分析書

## 目的

本ドキュメントは、整備工場の各入庫区分（車検、故障診断、修理・整備など）ごとに、実際の業務フローを分析し、「あるべき姿」を設定するための基礎資料です。

## 入庫区分一覧

現在のシステムで定義されている入庫区分：

1. **車検** - 法定24ヶ月点検
2. **故障診断** - 不具合の原因特定
3. **修理・整備** - 部品交換・メンテナンス
4. **12ヵ月点検** - 法定12ヶ月点検
5. **エンジンオイル交換** - 定期メンテナンス
6. **タイヤ交換・ローテーション** - タイヤ関連作業
7. **レストア** - 車両の復元・修復
8. **チューニング** - 性能向上・カスタマイズ
9. **パーツ取付** - アクセサリー・パーツ取り付け
10. **コーティング** - ボディコーティング・保護
11. **その他** - 上記に該当しない作業

## 1. 車検（法定24ヶ月点検）

### 1-1. 業務の特徴

- **所要日数**: 通常1日（午前入庫→午後出庫）
- **検査項目**: 法定項目（100項目以上）
- **作業内容**: 点検・検査が中心、必要に応じて部品交換
- **顧客の関心**: 「車検を通すこと」「費用」「時間」

### 1-2. 現在のプロセス（想定）

```
受付 → 診断 → 見積 → 作業 → 引渡
```

### 1-3. 実際の業務フロー（あるべき姿）

#### Phase 0: 予約・事前準備
- **顧客の行動**: 車検有効期限が近づいたら予約
- **スタッフの行動**: 車検証の確認、前回の整備履歴確認
- **システム要件**: 車検有効期限のリマインド、前回整備履歴の表示

#### Phase 1: 受付
- **所要時間**: 約5分
- **確認事項**: 
  - 車検証の確認（有効期限、車両情報）
  - 前回整備履歴の確認
  - 顧客の要望（任意点検項目の希望）
- **記録内容**: 
  - 走行距離（車検証記載）
  - 前回整備からの走行距離
  - 顧客の要望事項

#### Phase 2: 診断・検査
- **所要時間**: 約2-3時間
- **検査項目**: 
  - **法定項目（必須）**: 100項目以上の点検
    - ブレーキ系統（パッド、ディスク、フルード）
    - エンジン系統（オイル、冷却水、ベルト）
    - 電装系統（バッテリー、ライト、ウィンカー）
    - 足回り（タイヤ、サスペンション、アライメント）
    - 排気ガス検査
    - 車体検査（錆、損傷）
  - **任意項目（推奨）**: 顧客の要望に応じて
- **記録内容**: 
  - 各項目の状態（正常/注意/要交換）
  - 写真（不具合箇所）
  - 測定値（ブレーキパッド残量、タイヤ溝深さなど）
  - コメント（整備士の所見）

#### Phase 3: 見積作成
- **所要時間**: 約30分
- **作業内容**: 
  - 法定費用の計算（車検費用、重量税、自賠責保険など）
  - 必要部品の見積もり
  - 任意項目の提案
- **記録内容**: 
  - 法定費用の内訳
  - 部品代・工賃の内訳
  - 写真・コメントの紐付け

#### Phase 4: 顧客承認
- **所要時間**: 顧客の検討時間による（即決〜数日）
- **顧客の行動**: 
  - 見積内容の確認
  - 任意項目の選択
  - 承認・却下
- **システム要件**: 
  - 法定項目と任意項目の明確な区分
  - 承認待ちの状態管理

#### Phase 5: 作業実施
- **所要時間**: 承認後、約2-4時間
- **作業内容**: 
  - 法定点検の実施
  - 承認された部品交換
  - 車検証の更新手続き
- **記録内容**: 
  - 実施した作業の記録
  - Before/After写真
  - 交換部品の記録

#### Phase 6: 引渡・説明
- **所要時間**: 約15-30分
- **作業内容**: 
  - 作業内容の説明
  - 車検証の返却
  - 次回点検の案内
- **記録内容**: 
  - 引渡日時
  - 次回点検予定日

### 1-4. システム要件（あるべき姿）

#### 診断画面での要件
- **検査項目の自動表示**: 車検用の100項目以上のチェックリスト
- **測定値の入力**: ブレーキパッド残量、タイヤ溝深さなど数値入力
- **法定/任意の区分**: 各項目が法定か任意かを明示
- **写真撮影**: 不具合箇所の証拠写真
- **コメント入力**: 整備士の所見（音声入力対応）

#### 見積画面での要件
- **法定費用の自動計算**: 車検費用、重量税、自賠責保険
- **部品代・工賃の入力**: 承認された項目のみ
- **写真・コメントの紐付け**: 各見積項目に証拠写真を紐付け

#### 作業画面での要件
- **作業項目の表示**: 承認された項目のみ表示
- **Before/After写真**: 交換部品の証拠写真
- **車検証更新**: 更新日時の記録

## 2. 故障診断

### 2-1. 業務の特徴

- **所要日数**: 1日〜数日（原因特定の難易度による）
- **検査項目**: 不具合箇所に応じて変動
- **作業内容**: 原因特定が中心、必要に応じて修理
- **顧客の関心**: 「何が原因か」「修理費用」「修理期間」

### 2-2. 実際の業務フロー（あるべき姿）

#### Phase 0: 予約・事前準備
- **顧客の行動**: 不具合を感じたら予約
- **スタッフの行動**: 不具合内容の確認、前回整備履歴の確認
- **システム要件**: 不具合内容の事前入力、前回整備履歴の表示

#### Phase 1: 受付
- **所要時間**: 約5-10分
- **確認事項**: 
  - 不具合の詳細（いつから、どんな症状、頻度など）
  - 走行距離
  - 前回整備からの経過
- **記録内容**: 
  - 不具合の詳細（顧客の説明）
  - 走行距離
  - 前回整備履歴

#### Phase 2: 診断・検査
- **所要時間**: 約1-3時間（原因特定の難易度による）
- **検査項目**: 
  - **症状に応じた検査**: 不具合箇所の特定
    - エンジン不調 → エンジン系統の検査
    - ブレーキ不調 → ブレーキ系統の検査
    - 異音 → 音源の特定
  - **関連箇所の検査**: 原因の可能性がある箇所
- **記録内容**: 
  - 検査結果（正常/異常/要交換）
  - 測定値（圧力、電圧、温度など）
  - 写真・動画（異音の録音、異常箇所の撮影）
  - コメント（整備士の所見、原因の推測）

#### Phase 3: 見積作成
- **所要時間**: 約30分-1時間
- **作業内容**: 
  - 原因特定の結果報告
  - 修理方法の提案
  - 費用見積もり
- **記録内容**: 
  - 原因の説明
  - 修理方法の提案
  - 部品代・工賃の見積もり
  - 写真・動画の紐付け

#### Phase 4: 顧客承認
- **所要時間**: 顧客の検討時間による（即決〜数日）
- **顧客の行動**: 
  - 原因の確認
  - 修理方法の確認
  - 費用の確認
  - 承認・却下
- **システム要件**: 
  - 原因の明確な説明
  - 修理方法の選択肢
  - 緊急度の表示（安全に関わる場合は強調）

#### Phase 5: 作業実施
- **所要時間**: 承認後、約1-4時間（修理内容による）
- **作業内容**: 
  - 原因箇所の修理
  - 関連箇所の点検・調整
  - 動作確認
- **記録内容**: 
  - 実施した修理の記録
  - Before/After写真
  - 交換部品の記録
  - 動作確認の結果

#### Phase 6: 引渡・説明
- **所要時間**: 約15-30分
- **作業内容**: 
  - 修理内容の説明
  - 動作確認のデモ
  - 今後の注意事項
- **記録内容**: 
  - 引渡日時
  - 顧客への説明内容

### 2-3. システム要件（あるべき姿）

#### 診断画面での要件
- **症状に応じた検査項目の表示**: 不具合内容から関連する検査項目を自動表示
- **動画撮影**: 異音の録音、動作不良の動画撮影
- **測定値の入力**: 圧力、電圧、温度など専門的な測定値
- **原因の推測**: 整備士の所見を記録
- **緊急度の設定**: 安全に関わる場合は緊急度を設定

#### 見積画面での要件
- **原因の明確な説明**: 写真・動画を交えた説明
- **修理方法の選択肢**: 複数の修理方法を提案可能
- **緊急度の表示**: 安全に関わる場合は強調表示

#### 主な違い

- **法定費用**: 車検費用なし（点検費用のみ）
- **所要時間**: 車検より短い（約1-2時間）

### 4-3. システム要件（あるべき姿）

車検とほぼ同様だが、検査項目が少ない。
