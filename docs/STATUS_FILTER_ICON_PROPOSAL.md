# ステータス別クイックフィルタ アイコン追加提案

## 現在の実装

- テキストのみのボタン
- バッジで件数を表示
- シンプルで機能的なデザイン

## アイコン追加のメリット・デメリット

### メリット

1. **視覚的識別性の向上**
   - 各ステータスをアイコンで素早く識別可能
   - テキストを読まなくても意味が伝わる

2. **デザインの一貫性**
   - JobCardのアクションボタンと統一感が出る
   - アプリケーション全体のデザイン言語が統一される

3. **スキャンしやすさ**
   - ユーザーが画面をスキャンする際、アイコンが視覚的なランドマークとして機能

### デメリット

1. **スペースの制約**
   - ボタンが小さくなり、テキストとアイコンの両方を配置すると窮屈になる可能性
   - モバイル表示で特に問題になる可能性

2. **視覚的ノイズ**
   - アイコンが多すぎると、かえって見づらくなる可能性
   - シンプルなデザインの方が良い場合もある

3. **保守性**
   - アイコンの選択と一貫性の維持が必要

## 推奨アイコン案

### 案1: 最小限のアイコン（推奨）

| ステータス | アイコン | 理由 |
|-----------|---------|------|
| すべて | `Grid` または `List` | 全体を表す |
| 入庫待ち | `Clock` または `Package` | 待機状態を表す |
| 診断待ち | `Activity` | 診断アクティビティを表す（JobCardと統一） |
| 見積作成待ち | `FileText` または `Calculator` | 見積書を表す（JobCardと統一） |
| 作業待ち | `Wrench` | 作業を表す（JobCardと統一） |
| 引渡待ち | `Car` または `CheckCircle2` | 引渡しを表す（JobCardと統一） |

### 案2: アイコンなし（現状維持）

- シンプルで機能的なデザインを維持
- テキストとバッジのみで十分に機能している

## UI/UXベストプラクティスに基づく推奨

### 推奨: **アイコンを追加する（案1）**

**理由**:
1. **一貫性**: JobCardのアクションボタンと統一感が出る
2. **視認性**: アイコンがあることで、各ステータスを素早く識別できる
3. **モダンなUI**: 現代のWebアプリケーションでは、アイコンとテキストの組み合わせが標準的

**実装時の注意点**:
- アイコンサイズは `h-4 w-4` 程度（小さめ）
- アイコンとテキストの間隔は `gap-1.5` 程度
- モバイル表示でも読みやすさを確保
- アイコンは `lucide-react` から統一して使用

## 実装例

```tsx
import { Grid, Clock, Activity, FileText, Wrench, Car } from "lucide-react";

const statusIcons = {
  すべて: Grid,
  入庫待ち: Clock,
  診断待ち: Activity,
  見積作成待ち: FileText,
  作業待ち: Wrench,
  引渡待ち: Car,
};

// ボタン内で使用
<Button>
  <Icon className="h-4 w-4" />
  {status.label}
  <Badge>{count}</Badge>
</Button>
```
