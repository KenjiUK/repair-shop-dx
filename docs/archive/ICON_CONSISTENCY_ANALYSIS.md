# アイコン一貫性分析と改善提案

## 現状分析

### TOPページ（JobCard）のアクションボタン

| ステータス | ボタンラベル | アイコン | 遷移先 |
|-----------|------------|---------|--------|
| 入庫待ち | 受付を開始 | `Key` | チェックイン処理 |
| 入庫済み | 診断を開始 | `Activity` | `/mechanic/diagnosis/[id]` |
| 見積作成待ち | 見積を開始 | `FileText` | `/admin/estimate/[id]` |
| 作業待ち | 作業を開始 | `Wrench` | `/mechanic/work/[id]` |
| 出庫待ち | 引渡しを開始 | `Car` | `/presentation/[id]` |

### 下層ページのタイトルとアイコン

| ページ | タイトル | アイコン | 備考 |
|-------|---------|---------|------|
| 診断ページ | 動的（例: "車検診断"） | `AlertCircle` | 内容によってタイトルが変わる |
| 作業ページ | 動的（例: "車検作業"） | `Wrench` | 内容によってタイトルが変わる |
| 見積ページ | "見積作成" | `Calculator` | 固定 |
| 引き渡しページ | "整備完了レポート" | なし | アイコンなし |

### 不一致の詳細

1. **診断ページ**
   - TOPページ: `Activity`（アクティビティ/診断中）
   - 下層ページ: `AlertCircle`（警告/注意）
   - **不一致**: アイコンが異なる

2. **見積ページ**
   - TOPページ: `FileText`（ファイル/文書）
   - 下層ページ: `Calculator`（計算機）
   - **不一致**: アイコンが異なる

3. **作業ページ**
   - TOPページ: `Wrench`（レンチ/工具）
   - 下層ページ: `Wrench`（レンチ/工具）
   - **一致**: アイコンが統一されている ✅

4. **引き渡しページ**
   - TOPページ: `Car`（車）
   - 下層ページ: アイコンなし
   - **不一致**: 下層ページにアイコンがない

## UX観点からの分析

### アイコン統一のメリット

1. **認知的一貫性**
   - ユーザーが同じアイコンを見て、同じ機能を連想できる
   - 学習コストの削減
   - ナビゲーションの理解が容易

2. **視覚的階層の明確化**
   - TOPページのボタン → 下層ページのタイトルという流れが視覚的に明確
   - 現在どのページにいるかが直感的に分かる

3. **ブランド一貫性**
   - デザインシステムとしての統一感
   - プロフェッショナルな印象

### 作業内容による違いの考慮

下層ページでは、作業内容によってタイトルが動的に変わります：

- **診断ページ**: "車検診断"、"エンジンオイル交換診断"、"タイヤ交換・ローテーション診断"など
- **作業ページ**: "車検作業"、"エンジンオイル交換作業"など

この場合、**アイコンは統一しつつ、タイトルで違いを表現する**のが適切です。

## 改善提案

### 提案1: 完全統一（推奨）

TOPページのアイコンを下層ページに合わせる。

| 機能 | 統一アイコン | 理由 |
|------|------------|------|
| 診断 | `Activity` → `AlertCircle` に変更 | 診断は「確認・点検」の意味で`AlertCircle`が適切 |
| 見積 | `FileText` → `Calculator` に変更 | 見積は「計算・見積もり」の意味で`Calculator`が適切 |
| 作業 | `Wrench`（維持） | 既に統一されている |
| 引き渡し | `Car`（下層ページに追加） | 引き渡しページに`Car`アイコンを追加 |

**メリット:**
- 完全な一貫性
- 下層ページのアイコンが既に適切な意味を持っている

**デメリット:**
- TOPページのアイコンを変更する必要がある

### 提案2: 下層ページをTOPページに合わせる

下層ページのアイコンをTOPページに合わせる。

| 機能 | 統一アイコン | 理由 |
|------|------------|------|
| 診断 | `AlertCircle` → `Activity` に変更 | TOPページの`Activity`に統一 |
| 見積 | `Calculator` → `FileText` に変更 | TOPページの`FileText`に統一 |
| 作業 | `Wrench`（維持） | 既に統一されている |
| 引き渡し | `Car`（下層ページに追加） | TOPページの`Car`に統一 |

**メリット:**
- TOPページの変更が不要
- 既存のユーザー体験を維持

**デメリット:**
- `Activity`は「診断」よりも「アクティビティ」の意味が強い
- `FileText`は「見積」よりも「文書」の意味が強い

### 提案3: 意味的により適切なアイコンに統一（推奨）

両方のページで、意味的により適切なアイコンを使用。

| 機能 | 統一アイコン | 理由 |
|------|------------|------|
| 診断 | `Activity`（両方で使用） | 「診断中・アクティブな作業」の意味 |
| 見積 | `Calculator`（両方で使用） | 「計算・見積もり」の意味が明確 |
| 作業 | `Wrench`（維持） | 既に統一されている |
| 引き渡し | `Car`（両方で使用） | 「車両の引き渡し」の意味が明確 |

**メリット:**
- 意味的に最も適切
- 完全な一貫性

**デメリット:**
- TOPページのアイコンを変更する必要がある

## 推奨案: 提案3（意味的により適切なアイコンに統一）

### 実装方針

1. **診断**: `Activity`に統一
   - TOPページ: `Activity`（維持）
   - 下層ページ: `AlertCircle` → `Activity`に変更

2. **見積**: `Calculator`に統一
   - TOPページ: `FileText` → `Calculator`に変更
   - 下層ページ: `Calculator`（維持）

3. **作業**: `Wrench`（維持）
   - 既に統一されている

4. **引き渡し**: `Car`に統一
   - TOPページ: `Car`（維持）
   - 下層ページ: アイコンなし → `Car`を追加

### タイトルの動的表示について

下層ページでは、作業内容によってタイトルが動的に変わりますが、**アイコンは統一**します。

**例:**
- 診断ページ: `Activity` + "車検診断" / "エンジンオイル交換診断" / "タイヤ交換・ローテーション診断"
- 作業ページ: `Wrench` + "車検作業" / "エンジンオイル交換作業"

これにより、**アイコンで機能を識別し、タイトルで具体的な作業内容を識別**できます。
