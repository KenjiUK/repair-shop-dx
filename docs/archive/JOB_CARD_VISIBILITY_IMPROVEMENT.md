# JOBカードの視認性改善提案

## 現在の問題

セクション全体に白背景を適用したことで、以下の問題が発生：

1. **セクション背景**: `bg-white`（白）
2. **JOBカード背景**: `bg-card`（白、デフォルト）
3. **結果**: セクションとカードが同じ白背景で、境界が不明瞭

## 解決策の検討

### 提案A: JOBカードの背景色を変更（推奨）

**実装:**
- JOBカードの背景色を`bg-slate-50`に変更
- セクション背景（`bg-white`）と対比して、カードが明確に区別される

**メリット:**
- セクションとカードの階層が明確になる
- 視認性が大幅に向上
- 2025年のベストプラクティス（階層的な背景色）に合致

**デザイン階層:**
```
ページ背景: bg-slate-50（薄いグレー）
  └─ セクション: bg-white（白）
      └─ JOBカード: bg-slate-50（薄いグレー）
```

### 提案B: JOBカードのシャドウとボーダーを強化

**実装:**
- JOBカードのシャドウを`shadow-md`から`shadow-lg`に変更
- ボーダーを`border-slate-300`から`border-slate-400`に変更（より濃い色）

**メリット:**
- 背景色を変更せずに視認性を向上
- 既存のデザインシステムを維持

**デメリット:**
- 提案Aほど明確な階層は得られない

### 提案C: セクションの背景色を変更（非推奨）

**実装:**
- セクション背景を`bg-slate-50`に戻す

**デメリット:**
- 提案Aの意図（セクションをカードとして認識）が失われる
- ページ全体の階層が不明確になる

## 推奨実装

**提案A（JOBカードの背景色を`bg-slate-50`に変更）を推奨**

理由:
1. セクションとカードの階層が明確になる
2. 視認性が大幅に向上
3. 2025年のベストプラクティス（階層的な背景色）に合致
4. セクションをカードとして認識する意図を維持

## 実装詳細

### JOBカードのスタイル変更

```tsx
<Card className="transition-all hover:shadow-md bg-slate-50">
  {/* ... */}
</Card>
```

または、`Card`コンポーネント自体を変更せず、`JobCard`コンポーネントで上書き：

```tsx
<Card className="transition-all hover:shadow-md bg-slate-50 border-slate-200">
  {/* ... */}
</Card>
```

### 視覚的な効果

**改善前:**
- セクション: `bg-white`
- JOBカード: `bg-white`
- 結果: 境界が不明瞭

**改善後:**
- セクション: `bg-white`
- JOBカード: `bg-slate-50`
- 結果: 境界が明確、階層が分かりやすい
