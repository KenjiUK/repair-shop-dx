# 検索バー配置ロジック分析（2025年ベストプラクティス）

## 現在の配置ロジック

### 現在の位置
- **サマリーセクションの下、車両一覧の上**
- 独立したセクションとして配置

### 現在のロジックの意図
1. **フィルター適用前の検索**: サマリーでフィルターを適用する前に検索できる
2. **独立した機能**: 検索は独立した機能として明確に表示
3. **視覚的な分離**: サマリーと車両一覧の間に配置

## 問題点

### 1. **コンテキストの不一致**
- 検索バーが車両一覧のコンテンツを検索するのに、車両一覧セクションの外にある
- ユーザーが「車両一覧を検索したい」と思った時に、検索バーが別の場所にある

### 2. **視覚的な関連性の欠如**
- 検索バーと車両一覧が視覚的に分離されている
- 検索が車両一覧に適用されることが明確でない

### 3. **画面スペースの無駄**
- 独立したセクションとして配置されているため、画面スペースを占有

## 2025年ベストプラクティス

### 1. **セクションヘッダー内統合**
- 検索バーをセクションヘッダー内に統合
- 検索対象のコンテンツと視覚的に関連付け
- 画面スペースの節約

### 2. **コンテキストに応じた配置**
- 検索対象のコンテンツの近くに配置
- ユーザーの期待に合致（「車両一覧を検索したい」→ 車両一覧セクション内に検索バー）

### 3. **Progressive Disclosure**
- セクションが展開されている時のみ検索バーを表示
- 折りたたみ時は非表示で画面スペースを節約

## 提案

### 提案A: 車両一覧ヘッダー内統合（推奨）

**実装:**
- 車両一覧セクションのタイトルバーの下に検索バーを配置
- セクションが展開されている時のみ表示
- 検索バーと車両一覧が視覚的に関連付けられる

**メリット:**
- コンテキストに応じた配置
- 画面スペースの節約
- 視覚的な関連性の向上
- 2025年ベストプラクティスに合致

### 提案B: 車両一覧の下に配置

**実装:**
- 車両一覧セクションの下に検索バーを配置
- 一覧を見た後に検索する場合に便利

**メリット:**
- 一覧確認後の検索に適している
- ただし、検索してから一覧を見る場合には不自然

### 提案C: 現在の位置を維持

**実装:**
- サマリーセクションの下、車両一覧の上に配置を維持

**メリット:**
- フィルター適用前の検索に適している
- ただし、コンテキストの不一致がある

## 推奨実装

**提案A（車両一覧ヘッダー内統合）を推奨**

理由:
1. コンテキストに応じた配置（検索対象のコンテンツの近く）
2. 画面スペースの節約
3. 視覚的な関連性の向上
4. 2025年ベストプラクティスに合致
5. セクションが展開されている時のみ表示（Progressive Disclosure）
