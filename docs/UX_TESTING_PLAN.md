## 1. テスト環境のセットアップ

### 1.1 ローカル開発環境の起動

**前提条件:**
- Node.js 18以上がインストールされていること
- 必要な依存関係がインストールされていること（`npm install`完了）

**起動手順:**
1. プロジェクトルートディレクトリに移動
   ```bash
   cd c:\Users\sjam0\Desktop\repair-shop-dx
   ```

2. 開発サーバーを起動
   ```bash
   npm run dev
   ```

3. ブラウザで `http://localhost:3000` にアクセスして動作確認

**確認事項:**
- ✅ 開発サーバーが正常に起動しているか
- ✅ トップページが正常に表示されるか
- ✅ エラーが発生していないか（ブラウザのコンソールを確認）

---

### 1.2 ブラウザ環境の要件

**推奨ブラウザ:**
- Google Chrome（最新版）
- Microsoft Edge（最新版）
- Firefox（最新版）

**必須機能:**
- 開発者ツール（F12）が使用可能
- スクリーンショット機能
- レスポンシブデザインモード（モバイル表示テスト用）

**推奨設定:**
- 画面解像度: 1920x1080以上（デスクトップ）
- 画面解像度: 375x667（モバイルテスト用）

---

### 1.3 テストデータの準備

#### 1.3.1 各役割のユーザーアカウント設定

**現在の実装状況:**
- 本システムは認証機能が未実装のため、全ユーザーが同じ権限でアクセス可能
- テスト時は、各役割の視点で操作を実施

**テスト時の注意:**
- 各役割のユーザーとして、該当する業務フローのみをテスト
- 役割ごとに重要視する情報が異なることを意識

---

#### 1.3.2 テスト案件の作成

**必要なテストデータ:**

1. **緊急案件（緊急度: 高）**
   - 緊急度インジケータ（赤い縦バー）が表示される案件
   - 複数のステータス（入庫待ち、診断待ち、作業待ちなど）で準備

2. **重要顧客の案件**
   - 重要顧客マーク（星アイコン）が表示される案件
   - 複数のステータスで準備

3. **各種入庫区分の案件**
   - 車検
   - 12ヵ月点検
   - 故障診断
   - 修理・整備
   - タイヤ交換・ローテーション
   - レストア
   - チューニング・パーツ取付
   - 板金・塗装
   - その他のメンテナンス
   - その他

4. **各種ステータスの案件**
   - 入庫待ち
   - 入庫済み
   - 診断待ち
   - 見積作成待ち
   - 見積提示済み
   - 作業待ち
   - 出庫待ち
   - 出庫済み

**テストデータの確認:**
- 現在のモックデータで上記の条件を満たす案件が存在するか確認
- 不足している場合は、モックデータを追加

   - スクリーンショット: `screenshots/scenario1-7-job-card.png`

- ✅ 一目で本日の業務量を把握できる

- ✅ フィルター機能で必要な情報に素早くアクセスできる

### 3.3 レビュー収集のタイムライン

**推奨スケジュール:**

1. **テスト実施後、即座にレビューを依頼**
   - 各ユーザーがテストを完了したら、すぐにレビューフォームを配布
   - 記憶が新しいうちにフィードバックを収集

2. **レビュー収集期限**
   - テスト実施から1週間以内にレビューを収集
   - 遅れる場合は、再度テストを実施してもらう

3. **レビュー結果の統合**
   - 全レビューを収集後、1週間以内に結果を統合
   - 問題点を優先順位付け
   - 改善提案を作成
