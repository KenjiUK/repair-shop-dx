## 概要

12ヶ月点検や24ヶ月点検（車検）などの定期メンテナンス系の作業では、**「いきなりの作業」ではなく、まず診断（検査）を行い、問題があれば見積もりを作成し、顧客に承認を得てから作業を実施する**という流れになっています。

これは、整備工場の業務フローとして一般的なもので、以下の理由があります：

1. **法定点検の性質**: 12ヶ月点検や24ヶ月点検（車検）は法定点検であり、多くの検査項目がある
2. **問題発見の可能性**: 検査中に問題（不具合）が見つかることがある
3. **見積もりへの反映**: 問題があれば、整備士が部品をリストアップし、見積もりに反映される
4. **顧客承認の必要性**: 見積もりを顧客に提示し、承認を得てから作業を実施する

---

## 12ヶ月点検・24ヶ月点検の性質

### 12ヶ月点検（法定12ヶ月点検）

**検査項目**: 29項目（5つのカテゴリ）
- かじ取り装置（ステアリング）
- 制動装置（ブレーキ）
- 走行装置
- 緩衝装置（サスペンション）
- 動力伝達装置（ドライブトレイン）

**所要日数**: その日のうちに終わるのがほとんど（問題があれば預かりになる）

**法定費用**: 車両重量に応じて決まっている（1500kg以下: ¥26,400、5000kg以下: ¥13,200 または ¥30,800）

### 24ヶ月点検（車検）

**検査項目**: 6つのカテゴリ
- エンジン・ルーム点検
- 室内点検
- 足廻り点検
- 下廻り点検
- 外廻り点検
- 日常点検

**所要日数**: 通常1日で完了（問題があれば預かりになる）

**法定費用**: 車両重量に応じて決まっている（例: 1500kg以下: ¥66,000）

## 診断フェーズの役割

### 1. 検査項目の入力

整備士が車両の状態を確認し、各検査項目の状態を入力します：

- **OK**: 問題なし
- **注意**: 軽微な問題（要観察）
- **要交換**: 交換が必要
- **調整**: 調整が必要
- **清掃**: 清掃が必要

### 2. 問題の発見と記録

検査中に問題（不具合）が見つかった場合：

- **写真・動画の撮影**: 不具合の証拠を撮影
- **コメントの入力**: 問題の詳細を記録
- **部品のリストアップ**: 交換が必要な部品をリストアップ

### 3. 見積もりへの反映

診断結果を基に、見積もりを作成します：

- **必須整備**: 法定費用など（固定ON、外せない）
- **推奨整備**: 整備士が「要交換」と判定したもの（デフォルトON、OFFにできる）
- **任意整備**: その他の整備（デフォルトOFF、ONにできる）

### 4. 顧客承認の取得

見積もりを顧客に提示し、承認を得ます：

- **顧客の選択**: 予算に合わせて、各作業をON/OFFで調整
- **リアルタイム計算**: 合計金額がリアルタイムで再計算される
- **承認確定**: 承認された作業のみが実施される

## 業務フローの流れ

### 標準的な流れ（問題なしの場合）

```
[Phase 1] 受付
    ↓
[Phase 2] 診断（検査項目の入力）
    ↓
[Phase 3] 見積作成（法定費用のみ）
    ↓
[Phase 4] 顧客承認（法定費用の承認）
    ↓
[Phase 5] 作業（法定点検の実施）
    ↓
[Phase 6] 報告（完了報告）
```

### 問題が見つかった場合の流れ

```
[Phase 1] 受付
    ↓
[Phase 2] 診断（検査項目の入力）
    ├─ 問題発見（例: ブレーキパッド残り2mm）
    ├─ 写真・動画の撮影
    ├─ コメントの入力
    └─ 部品のリストアップ（フロントブレーキパッド交換）
    ↓
[Phase 3] 見積作成
    ├─ 法定費用（必須整備）
    └─ 推奨整備（ブレーキパッド交換 ¥33,000）
    ↓
[Phase 4] 顧客承認
    ├─ 法定費用: 承認（固定ON）
    └─ ブレーキパッド交換: 承認（顧客が選択）
    ↓
[Phase 5] 作業
    ├─ 法定点検の実施
    └─ ブレーキパッド交換の実施
    ↓
[Phase 6] 報告（完了報告）
```

## なぜ「いきなりの作業」ではないのか

### 1. 法定点検の性質

12ヶ月点検や24ヶ月点検（車検）は**法定点検**であり、以下の特徴があります：

- **検査項目が多い**: 12ヶ月点検は29項目（5カテゴリ）、車検は6カテゴリ
- **法的な記録が必要**: PDFテンプレートに基づいて記録を残す必要がある
- **問題発見の可能性**: 検査中に問題（不具合）が見つかることがある

### 2. 問題発見の可能性

検査中に問題（不具合）が見つかることがあります：

- **ブレーキパッドの摩耗**: 残り2mm（要交換）
- **タイヤの溝深さ不足**: 1.6mm未満（要交換）
- **エアフィルターの汚れ**: 交換推奨
- **その他の不具合**: 各種部品の劣化

### 3. 顧客承認の必要性

問題があれば、顧客に説明し、見積もりを提示して承認を得る必要があります：

- **透明性**: 顧客に問題を説明し、写真・動画を見せて理解してもらう
- **選択の自由**: 顧客が予算に合わせて、作業を選択できる
- **法的な記録**: 承認された作業のみが実施され、記録に残る

### 4. 業務フローの統一

すべての入庫区分で同じ業務フローを適用することで：

- **一貫性**: すべての作業で同じフローを適用
- **品質管理**: 診断→見積→承認→作業という流れで品質を確保
- **記録の完全性**: すべての作業で診断結果と作業記録が残る

## 入庫区分別の診断の必要性

### 診断が必要な入庫区分

| 入庫区分 | 診断の必要性 | 理由 |
|---------|------------|------|
| **12ヶ月点検** | ✅ 必要 | 法定点検、29項目の検査が必要 |
| **24ヶ月点検（車検）** | ✅ 必要 | 法定点検、6カテゴリの検査が必要 |
| **故障診断** | ✅ 必要 | 問題の特定と原因の調査が必要 |
| **修理・整備** | ✅ 必要 | 問題の特定と修理内容の決定が必要 |
| **エンジンオイル交換** | ❌ 不要 | 単純な作業、診断不要 |
| **タイヤ交換・ローテーション** | ❌ 不要 | 単純な作業、診断不要 |
| **コーティング** | ❌ 不要 | 単純な作業、診断不要 |

### 診断が不要な入庫区分

以下の入庫区分は、診断フェーズをスキップして直接作業フェーズに進むことができます：

- **エンジンオイル交換**: 単純な作業、診断不要
- **タイヤ交換・ローテーション**: 単純な作業、診断不要
- **コーティング**: 単純な作業、診断不要

**ただし、これらの作業も12ヶ月点検や車検と同時に実施する場合は、診断フェーズで一緒に管理されます。**

## 実装上の考慮事項

### 1. 診断フェーズのスキップ

単純な作業（エンジンオイル交換など）の場合は、診断フェーズをスキップして直接作業フェーズに進むことができます。

**実装方法:**
- 入庫区分に応じて、診断フェーズをスキップするフラグを設定
- 診断フェーズをスキップする場合は、直接作業フェーズに進む

### 2. 複数作業の管理

1つの入庫に対して複数の作業（車検、オイル交換、コーティングなど）を管理する場合：

- **診断フェーズ**: 各作業ごとに診断を実施
- **見積フェーズ**: 各作業ごとに見積を作成
- **承認フェーズ**: 各作業ごとに承認を取得
- **作業フェーズ**: 各作業ごとに作業を実施

### 3. 診断結果の再利用

同じ入庫内で複数の作業がある場合、診断結果を再利用できます：

- **例**: 12ヶ月点検とオイル交換を同時に実施する場合
  - 12ヶ月点検の診断結果を基に、オイル交換の必要性を判断
  - 診断結果を共有して、見積もりに反映

### 診断フェーズが必要な理由

1. **法定点検の性質**: 12ヶ月点検や24ヶ月点検（車検）は法定点検であり、多くの検査項目がある
2. **問題発見の可能性**: 検査中に問題（不具合）が見つかることがある
3. **見積もりへの反映**: 問題があれば、整備士が部品をリストアップし、見積もりに反映される
4. **顧客承認の必要性**: 見積もりを顧客に提示し、承認を得てから作業を実施する

### 業務フローの統一

すべての入庫区分で同じ業務フローを適用することで：

- **一貫性**: すべての作業で同じフローを適用

- **記録の完全性**: すべての作業で診断結果と作業記録が残る

### 将来の拡張性

診断フェーズをすべての入庫区分で統一することで：

- **新しい入庫区分の追加**: 新しい入庫区分を追加しても、同じフローを適用できる
