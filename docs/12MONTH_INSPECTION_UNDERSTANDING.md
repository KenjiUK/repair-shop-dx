# 12ヶ月点検 業務理解メモ

## ドキュメント情報

- **作成日**: 2025-01-XX
- **バージョン**: 1.0
- **対象**: 12ヶ月点検（法定12ヶ月点検）の業務理解
- **目的**: 車検と同様に、12ヶ月点検の業務フローを想像し、仕様を確定する

---

## 1. 12ヶ月点検の基本情報

### 1-1. 法令上の位置づけ

- **根拠法令**: 自動車点検基準（別表第6 - 自家用乗用自動車等）
- **点検頻度**: 1年ごと（12ヶ月ごと）
- **点検項目数**: 29項目（法定項目）
- **保存期間**: 2年間保存
- **OBD点検**: 令和3年10月1日から義務化（追加項目）

### 1-2. 点検項目の特徴

- **項目数**: 29項目（車検の100項目以上と比較して少ない）
- **カテゴリ**: 別表第6に基づく以下のカテゴリ
  1. かじ取り装置（ステアリング）
  2. 制動装置（ブレーキ）
  3. 走行装置
  4. 緩衝装置（サスペンション）
  5. 動力伝達装置（ドライブトレイン）
- **省略可能項目**: （※1）注記が付く項目は走行距離など条件により省略可能

### 1-3. 記録簿の構成

#### A. ヘッダー（車両・依頼者・作業情報）

**必須記載事項**:
- 自動車登録番号／車両番号／車台番号
- 点検（または特定整備）時の総走行距離
- 点検・整備を実施した者の氏名（名称）・住所
- 点検日／整備完了日
- 車名・型式・原動機型式・初度登録年月
- 依頼者（使用者）氏名・住所
- 工場名・認証番号、整備主任者名

**データソース**: Google Sheets「【DB】車両マスタ」から自動取得

#### B. チェックリスト（点検箇所×点検項目×結果記号）

**結果記号**:
- 該当なし
- 点検良好
- 調整
- 清掃
- 給油
- 締付
- 交換
- 修理
- （特定整備）

**カテゴリ**:
1. かじ取り装置（ステアリング）
2. 制動装置（ブレーキ）
3. 走行装置
4. 緩衝装置（サスペンション）
5. 動力伝達装置（ドライブトレイン）

#### C. 点検結果・整備概要（自由記述）

- 交換部品
- 作業内容
- 備考
- 特定整備が絡む場合: どの作業を誰がどこで実施したか（外注・離れた作業場等）

#### D. OBD点検（追加項目）

- 車載式故障診断装置の診断結果
- 令和3年10月1日から義務化
- **実装方法**: 別システムで実施、後程スタッフが診断結果をアップロード、顧客が確認可能

## 2. 12ヶ月点検の点検項目リスト（別表第6）

    - 10-1 ドラムとライニングとのすき間（※1）
    - 10-2 シュー摺動部分及びライニングの摩耗（※1）
    - 10-3 ドラムの摩耗及び損傷
11. **ブレーキ・ディスク及びパッド**:
    - 11-1 ディスクとパッドとのすき間（※1）
    - 11-2 パッドの摩耗（※1）
    - 11-3 ディスクの摩耗及び損傷

### 2-3. 走行装置

12. **ホイール**:
    - 12-1 タイヤの状態（※1）
    - 12-2 ホイール・ナット／ホイール・ボルトの緩み（※1）

### 2-5. 動力伝達装置（ドライブトレイン）

16. **トランスミッション及びトランスファ**: 油漏れ及び油量（※1）
17. **プロペラ・シャフト及びドライブ・シャフト**:
    - 17-1 連結部の緩み（※1）
    - 17-2 自在継手部ダスト・ブーツの亀き裂及び損傷
18. **デファレンシャル**: 油漏れ及び油量（※1）

19. **OBD点検**: 車載式故障診断装置の診断結果

## 3. 車検との比較

| 項目 | 車検（24ヶ月点検） | 12ヶ月点検 |
|------|-------------------|-----------|
| 点検頻度 | 2年ごと | 1年ごと |
| 点検項目数 | 100項目以上 | 29項目 |
| カテゴリ数 | 6カテゴリ | 5カテゴリ + OBD |
| 法定費用 | あり（車検費用、重量税等） | なし（点検料のみ） |
| 測定値 | CO・HC濃度、タイヤ溝深さ、ブレーキパッド厚さ | 必要に応じて |
| PDFテンプレート | 24か月点検用テンプレート.pdf | 要確認 |
| 所要日数 | 複数日（部品調達等） | 1日（標準） |

## 4. 確認済み事項

### 4-1. 業務フローについて

**質問1**: 12ヶ月点検の所要日数はどのくらいですか？

**回答**: ✅ **その日のうちに終わるのがほとんど。問題があれば預かりになる**

**設計決定**:
- 標準: 1日で完了（午前入庫→午後出庫）
- 問題がある場合: 預かり（複数日にわたる可能性あり）

**質問2**: 12ヶ月点検にも法定費用はありますか？

**回答**: ✅ **車両重量に応じて決まっている**

**法定費用（車両重量に応じた金額）**:
- 5000kg以下: ¥13,200
- 1500kg以下: ¥26,400
- 5000kg以下: ¥30,800

**設計決定**:
- Google Sheets「【DB】車両マスタ」内の「車両_同期用」シートから車両重量を取得
- 車両重量に応じて法定費用を自動計算
- または、Google Sheetsに「12ヶ月点検費用」の項目があれば、そこから取得

### 4-2. 検査項目について

**質問3**: 12ヶ月点検の検査項目は、車検と同じようにカテゴリごとに分けて入力しますか？

**回答**: ✅ **項目が多いから分けたほうがいい**

**設計決定**:
- 5つのカテゴリ（かじ取り装置、制動装置、走行装置、緩衝装置、動力伝達装置）ごとに分けて入力
- タブ形式でカテゴリを切り替え
- 各カテゴリの完了状況を表示

**質問4**: 測定値の入力は必要ですか？

**回答**: ✅ **PDFテンプレート（12カ月点検テンプレート.pdf）を確認する必要がある**

**設計決定**:
- PDFテンプレート（12カ月点検テンプレート.pdf）を確認して、測定値の入力が必要かどうかを判断
- 一般的な12ヶ月点検では、タイヤ溝深さ、ブレーキパッド厚さなどの測定値が必要な場合がある
- 必要に応じて測定値入力画面を実装
- **要確認**: PDFテンプレートに測定値の入力欄があるかどうか

**質問5**: OBD点検はどのように実施しますか？

**回答**: ✅ **別システムなので、後程スタッフが出し診断結果をアップロードして顧客が見れるようにする**

**設計決定**:
- OBD診断は別システムで実施
- スタッフが診断結果をアップロード
- 顧客が診断結果を確認できるようにする
- 診断結果はPDFまたは画像形式でアップロード可能

### 4-3. 写真・動画について

**質問6**: 写真・動画の撮影は必要ですか？

**回答**: ✅ **車検と同じでいい**

**設計決定**:
- 不具合が見つかった項目のみ写真・動画撮影
- 動画は最大15秒、Google Driveに保存
- 車検と同じUI/UX

### 4-4. 部品リストアップについて

**質問7**: 診断時の部品リストアップは必要ですか？

**回答**: ✅ **問題があれば整備士がリストアップ必要**

**設計決定**:
- 車検と同様に、整備士が診断時に交換が必要な部品をリストアップ
- 音声入力対応、レコメンド機能
- 見積もり画面に自動表示

### 4-5. PDFテンプレートについて

**質問8**: 12ヶ月点検用のPDFテンプレートはありますか？

**回答**: ✅ **12カ月点検テンプレート.pdf**

**設計決定**:
- PDFテンプレート: `public/12カ月点検テンプレート.pdf`
- このテンプレートに基づいて検査項目を定義
- 最終的にPDFを生成

### 4-6. データソースについて

**質問9**: 車両情報や費用情報の取得元は車検と同じですか？

**回答**: ✅ **車両情報は同じスプレッドシート**

**設計決定**:
- Google Sheets「【DB】車両マスタ」から車両情報を取得
- 法定費用は車両重量に応じて自動計算、または「車両_同期用」シートから取得

## 5. 確定した業務フロー

### 5-1. 受付から診断まで

```
1. 顧客が来店（または予約）
2. フロントスタッフが受付
3. 整備士が診断を開始
   - 29項目の点検を実施（5つのカテゴリごと）
   - 必要に応じて測定値を入力（PDFテンプレートを確認）
   - 不具合が見つかった項目のみ写真・動画を撮影
   - OBD点検を実施（別システム、後で診断結果をアップロード）
   - 問題があれば、交換が必要な部品をリストアップ
4. 診断完了
```

### 5-2. 見積もりから承認まで

```
1. フロントスタッフが見積もりを作成
   - 法定費用を確認（車両重量に応じて自動計算）
   - 整備士がリストアップした部品に金額を入力（該当する場合）
   - OBD診断結果をアップロード（別システムから）
2. 顧客に送信
3. 顧客が承認
```

### 5-3. 作業から出庫まで

```
1. 整備士が作業を実施
   - 承認された作業項目のみ実施
   - Before/After写真を撮影
   - 交換部品を記録
2. 作業完了
3. PDFを生成（12ヶ月点検記録簿）
4. フロントスタッフが引渡し・説明
5. 出庫完了（その日のうちに完了することが多い）
```

## 6. 確定した仕様サマリー

### 6-1. 業務フロー

- **所要日数**: その日のうちに終わるのがほとんど（問題があれば預かり）
- **法定費用**: 車両重量に応じて決まっている（5000kg以下: ¥13,200、1500kg以下: ¥26,400、5000kg以下: ¥30,800）
- **検査項目**: 29項目（5つのカテゴリごとに入力）
- **OBD点検**: 別システムで実施、後で診断結果をアップロード
- **写真・動画**: 車検と同じ（不具合項目のみ撮影）
- **部品リストアップ**: 問題があれば整備士がリストアップ

### 6-2. データソース

- **車両情報**: Google Sheets「【DB】車両マスタ」から取得
- **法定費用**: 車両重量に応じて自動計算、または「車両_同期用」シートから取得
- **PDFテンプレート**: `public/12カ月点検テンプレート.pdf`

### 6-3. 次のステップ

1. **PDFテンプレートの確認**: 12カ月点検テンプレート.pdfを確認して、測定値の入力が必要かどうかを判断
2. **詳細設計の作成**: 車検と同様に、詳細設計を作成
3. **ユーザージャーニーの作成**: 車検と同様に、ユーザー視点での動きを詳細にドキュメント化
4. **画面設計**: 診断画面、見積もり画面、作業画面の詳細設計
