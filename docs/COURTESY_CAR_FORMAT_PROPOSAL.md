# 代車情報表示フォーマット改善提案

## 📋 概要

代車在庫カードの詳細ダイアログを、車両詳細情報と同じフォーマットに統一する提案です。

## 🔍 現在の実装の分析

### 車両詳細情報のフォーマット

**構造:**
- グリッドレイアウト（`grid grid-cols-2`）
- ラベルと値のペア
- アイコン付きラベル
- セクション分け（車両マスタ情報、代車情報）

**表示項目:**
- 代車ID
- 車名
- ナンバープレート
- ステータス（バッジ表示）
- 紐付け中の案件ID
- 貸出開始日時

**UIデザイン:**
```
┌─────────────────────────────┐
│ 代車情報              [空き] │
├─────────────────────────────┤
│ 代車ID    │ 車名            │
│ CAR-001   │ トヨタ プリウス  │
├───────────┴─────────────────┤
│ ナンバープレート              │
│ 品川 500 あ 1234            │
├─────────────────────────────┤
│ 紐付け中の案件ID             │
│ job-12345                    │
├─────────────────────────────┤
│ 貸出開始日時                 │
│ 2024/12/01 10:00            │
└─────────────────────────────┘
```

### 現在の代車在庫カードのフォーマット

**構造:**
- ステータス別にグループ化（空き、使用中、点検中）
- シンプルなリスト表示

**表示項目:**
- 車名
- ナンバー（簡易表示）

**UIデザイン:**
```
┌─────────────────────────────┐
│ 空き (3台)                  │
├─────────────────────────────┤
│ トヨタ プリウス             │
│ ナンバー: 品川 500 あ 1234  │
├─────────────────────────────┤
│ ホンダ フィット             │
│ ナンバー: 品川 500 い 5678  │
└─────────────────────────────┘
```

## 🎯 改善提案

### 提案1: 車両詳細情報と同じフォーマット（推奨）⭐

**実装方法:**
- 各代車を個別のカードとして表示
- グリッドレイアウト（`grid grid-cols-2`）で情報を整理
- 車両詳細情報と同じ項目を表示

**表示項目:**
- 代車ID
- 車名
- ナンバープレート
- ステータス（バッジ表示）
- 紐付け中の案件ID（使用中の場合）
- 貸出開始日時（使用中の場合）

**メリット:**
- ✅ 車両詳細情報と統一されたフォーマット
- ✅ 情報の見やすさが向上
- ✅ 一貫性のあるUI/UX
- ✅ 40歳以上ユーザーにとって分かりやすい

**デメリット:**
- ⚠️ 画面スペースを多く使用
- ⚠️ スクロールが必要になる可能性

### 提案2: ハイブリッド形式

**実装方法:**
- ステータス別にグループ化は維持
- 各代車をカード形式で表示
- グリッドレイアウトで情報を整理

**メリット:**
- ✅ ステータス別のグループ化が維持される
- ✅ 情報の見やすさが向上
- ✅ 車両詳細情報と統一されたフォーマット

**デメリット:**
- ⚠️ 実装がやや複雑

### 提案3: コンパクト形式（現在の形式を改善）

**実装方法:**
- 現在の形式を維持
- 表示項目を追加（代車ID、貸出開始日時など）
- レイアウトを改善

**メリット:**
- ✅ 実装が簡単
- ✅ 画面スペースを効率的に使用

**デメリット:**
- ⚠️ 車両詳細情報とフォーマットが異なる
- ⚠️ 情報の見やすさが限定的
