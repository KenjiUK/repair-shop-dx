# 入庫区分別 業務理解メモ

## 目的

本ドキュメントは、各入庫区分ごとに、システム開発者が業務をどのように理解しているかを記録し、実際の業務との整合性を確認するための資料です。

---

## 1. 車検（法定24ヶ月点検）

### 1-1. 私の理解：車検とは何か

車検とは、**自動車の安全を確保するために、2年ごと（新車は3年）に法律で義務付けられている検査**です。

- **正式名称**: 自動車検査登録制度
- **目的**: 自動車が安全に走行できる状態かを確認
- **有効期限**: 通常2年間（新車は3年間）
- **実施場所**: 国土交通省の検査場、または整備工場（代行検査）

### 1-2. 私の理解：車検の業務フロー

#### 顧客の視点から見た流れ

1. **予約**
   - 車検有効期限が近づいたら、整備工場に予約
   - 電話、Web、LINEなどで予約
   - 「車検お願いします」と伝える

2. **来店・受付**
   - 指定日時に来店
   - 車検証を持参
   - 受付で車検証を提示
   - スタッフが車検証を確認

3. **検査・診断**
   - 整備工場で車両を検査
   - 法定項目（100項目以上）を点検
   - 不具合があれば指摘
   - 顧客は待つ（2-3時間）

4. **見積もり提示**
   - 検査結果を説明
   - 必要なら部品交換の見積もり
   - 法定費用（車検費用、重量税、自賠責保険など）の説明
   - 顧客が承認

5. **作業実施**
   - 承認後、作業を実施
   - 部品交換、調整など
   - 車検証の更新手続き

6. **引渡し**
   - 作業完了後、車検証を返却
   - 作業内容の説明
   - 次回点検の案内

#### スタッフ（フロント）の視点から見た流れ

1. **予約受付**
   - 顧客から「車検お願いします」と予約
   - 車検有効期限を確認
   - 前回整備履歴を確認
   - 予約日時を決定

2. **来店・受付**
   - 顧客が来店
   - 車検証を確認
   - 車検証の有効期限、車両情報を確認
   - 前回整備からの走行距離を確認
   - 顧客の要望を確認（任意点検項目の希望など）
   - システムに「受付」を記録

3. **整備士への引き継ぎ**
   - 鍵を整備士に渡す
   - 顧客の要望を伝える
   - 前回整備履歴を伝える

4. **見積もり作成（整備士から報告を受けた後）**
   - 整備士からの診断結果を確認
   - 法定費用を計算
   - 必要部品の見積もりを作成
   - 顧客に説明

5. **顧客承認待ち**
   - 顧客が検討
   - 承認を待つ

6. **作業完了確認**
   - 整備士から作業完了の報告
   - 車検証の更新確認

7. **引渡し**
   - 顧客に車検証を返却
   - 作業内容を説明
   - 次回点検の案内

#### スタッフ（整備士）の視点から見た流れ

1. **車両の受け取り**
   - フロントから鍵を受け取る
   - 顧客の要望を確認
   - 前回整備履歴を確認

2. **検査・診断**
   - 車両をピットに入れる
   - **法定項目（100項目以上）を点検**
     - ブレーキ系統: パッド、ディスク、フルード、ホース
     - エンジン系統: オイル、冷却水、ベルト、エアフィルター
     - 電装系統: バッテリー、ライト、ウィンカー、ワイパー
     - 足回り: タイヤ、サスペンション、アライメント
     - 排気ガス: 排気ガス検査
     - 車体: 錆、損傷、ナンバープレート
   - **測定値の記録**
     - ブレーキパッド残量（mm）
     - タイヤ溝深さ（mm）
     - バッテリー電圧（V）
     - その他、測定可能な項目
   - **写真撮影**
     - 不具合箇所の写真
     - 測定値の証拠写真
   - **コメント入力**
     - 各項目の状態（正常/注意/要交換）
     - 整備士の所見

3. **診断結果の報告**
   - 診断結果をフロントに報告
   - 必要部品をリストアップ

4. **作業実施（顧客承認後）**
   - 承認された項目のみ作業
   - 部品交換、調整など
   - Before/After写真を撮影
   - 作業内容を記録

5. **車検証更新手続き**
   - 車検証の更新手続き（代行検査の場合）
   - 更新日時を記録

6. **作業完了報告**
   - フロントに作業完了を報告

### 1-3. 私の理解：車検で重要なポイント

#### 法定項目と任意項目の区分

- **法定項目（必須）**: 車検を通すために必ず必要な項目
  - 例: ブレーキパッドが残り2mm以下 → 交換必須
  - 例: タイヤの溝が1.6mm以下 → 交換必須
  - 例: 排気ガスが基準値超過 → 修理必須

- **任意項目（推奨）**: 車検は通るが、安全のため推奨される項目
  - 例: ブレーキパッドが残り3mm → まだ使えるが、交換推奨
  - 例: エアフィルターが汚れている → 交換推奨

#### 法定費用の内訳

- **車検費用**: 検査費用（約17,000円）
- **重量税**: 車両の重量に応じた税金（約30,000円〜）
- **自賠責保険**: 強制保険（約25,000円）
- **印紙代**: 車検証の印紙代（約1,000円）

これらは**車両の種類・重量によって自動的に決まる**ため、システムで自動計算できる。

#### 所要時間

- **標準**: 1日（午前入庫→午後出庫）
  - 検査: 2-3時間
  - 見積もり: 30分
  - 顧客承認: 30分-1時間（即決が多い）
  - 作業: 2-4時間（承認された作業による）
  - 合計: 約5-8時間

- **複数日**: 部品調達が必要な場合
  - 例: 特殊な部品が必要で、在庫がない場合
  - 例: 大きな修理が必要な場合

#### 記録すべき内容

1. **診断時**
   - 各項目の状態（正常/注意/要交換）
   - 測定値（ブレーキパッド残量、タイヤ溝深さなど）
   - 写真（不具合箇所）
   - コメント（整備士の所見）

2. **見積もり時**
   - 法定費用の内訳
   - 部品代・工賃の内訳
   - 各項目に写真・コメントを紐付け

3. **作業時**
   - 実施した作業の記録
   - Before/After写真
   - 交換部品の記録
   - 車検証更新日時

### 1-4. 私の理解：システムで実現すべきこと

#### 診断画面で

1. **車検用の検査項目を自動表示**
   - 100項目以上のチェックリスト
   - 各項目が「法定」か「任意」かを表示
   - 項目ごとに状態を選択（正常/注意/要交換）

2. **測定値の入力**
   - ブレーキパッド残量（mm）
   - タイヤ溝深さ（mm）
   - バッテリー電圧（V）
   - その他、測定可能な項目

3. **写真撮影**
   - 不具合箇所の写真
   - 測定値の証拠写真（例: ブレーキパッド残量を定規で測っている写真）

4. **コメント入力**
   - 各項目の所見
   - 音声入力対応（手が汚れているため）

#### 見積画面で

1. **法定費用の自動計算**
   - 車両の種類・重量から自動計算
   - 車検費用、重量税、自賠責保険、印紙代

2. **部品代・工賃の入力**
   - 承認された項目のみ
   - 各項目に写真・コメントを紐付け

3. **法定/任意の区分表示**
   - 顧客に「これは必須です」「これは推奨です」と明確に表示

#### 作業画面で

1. **承認された項目のみ表示**
   - 顧客が承認した項目のみ作業
   - 見送られた項目は非表示またはグレーアウト

2. **Before/After写真**
   - 交換前の写真
   - 交換後の写真
   - 並べて比較表示

3. **車検証更新の記録**
   - 更新日時を記録
   - 次回車検予定日を自動計算（2年後）

### 1-5. 私の理解：不明点・確認したいこと

1. **検査項目の詳細**
   - 100項目以上の具体的な項目リストは？
   - 各項目の判定基準は？（例: ブレーキパッドが何mm以下で「要交換」か）
   - 測定値が必要な項目はどれか？

2. **法定費用の計算方法**
   - 車両の種類・重量からどのように計算するか？
   - システムに車両情報（重量など）はあるか？

3. **車検証更新手続き**
   - 代行検査の場合、システムでどのように管理するか？
   - 更新日時の記録方法は？

4. **複数日の場合の管理**
   - 部品調達が必要な場合、どのように管理するか？
   - 顧客に連絡するタイミングは？

5. **前回整備履歴の活用**
   - 前回整備からの走行距離はどのように確認するか？
   - 前回の診断結果と比較する機能は必要か？
