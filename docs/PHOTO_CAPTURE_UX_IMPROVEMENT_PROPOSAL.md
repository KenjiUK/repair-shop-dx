# 写真撮影UX/UI改善提案

## 現状の問題点

### 現在のフロー
1. ユーザーが「交換」「修理」などのステータスを選択
2. 「写真を撮影しますか？」という確認ダイアログが表示
3. 「はい」を選択すると、写真撮影UIが表示される
4. その後、追加の写真アイコン（+）が表示される
5. そのアイコンをクリックすると、ボトムシートで「写真を撮影」と「動画を録画」の選択肢が表示される

### 問題点
1. **二段階の確認が複雑**: ダイアログ → ボトムシートの二段階で、ユーザーの操作が増える
2. **UIの一貫性がない**: ダイアログとボトムシートのデザインが異なり、統一感がない
3. **写真撮影のフローが分かりにくい**: 最初の確認で「はい」を選んだのに、再度追加ボタンを押す必要がある
4. **モーダルの種類が混在**: AlertDialogとSheetが混在しており、UXが一貫していない

## 改善提案

### 提案1: 確認ダイアログを削除し、直接ボトムシートを表示（推奨）

#### 新しいフロー
1. ユーザーが「交換」「修理」などのステータスを選択
2. **直接ボトムシートを表示**（確認ダイアログなし）
3. ボトムシートで以下の選択肢を提供：
   - **写真を撮影**（推奨・大きく表示）
   - **動画を録画**（オプション）
   - **スキップ**（写真撮影をスキップ）

#### メリット
- **操作ステップが1つ減る**: ダイアログ → ボトムシート → 撮影 → 追加ボタン → ボトムシート → 撮影（5ステップ）から、ステータス選択 → ボトムシート → 撮影（3ステップ）に削減
- **UIの一貫性**: すべてボトムシートで統一
- **直感的な操作**: ステータス選択後、すぐに撮影オプションが表示される
- **モバイルフレンドリー**: ボトムシートはモバイルで使いやすい

#### UIデザイン
```
┌─────────────────────────────┐
│  [項目名]                    │
│  残り X 項目                 │
├─────────────────────────────┤
│                             │
│  ┌─────────────────────┐   │
│  │ 📷 写真を撮影        │   │ ← 大きく、目立つ
│  └─────────────────────┘   │
│                             │
│  ┌──────────┐ ┌──────────┐│
│  │ 🎥 動画   │ │ ⏭️ スキップ││
│  └──────────┘ └──────────┘│
│                             │
└─────────────────────────────┘
```

### 提案2: 写真撮影後は自動的にメディアUIを表示

#### 改善点
- 写真撮影後、追加ボタンを押さなくても、撮影した写真がすぐに表示される
- 追加の写真を撮りたい場合は、表示された写真の横の「+」ボタンから追加可能

#### メリット
- **即座のフィードバック**: 撮影した写真がすぐに確認できる
- **操作の明確化**: 追加ボタンを探す必要がない

### 提案3: ボトムシートのデザイン統一

#### 統一すべき要素
1. **ボタンサイズ**: `h-16` (64px) で統一（良好ボタンと同じ）
2. **フォントサイズ**: `text-xl` (20px) / `text-base` (16px) で統一
3. **アイコンサイズ**: `h-6 w-6` (24px) で統一
4. **パディング**: `px-4 py-4` で統一
5. **角丸**: `rounded-lg` で統一

#### カラーシステム
- **写真撮影**: 緑系（`bg-green-600` / `text-green-700`）
- **動画録画**: 青系（`bg-blue-600` / `text-blue-700`）
- **スキップ**: グレー系（`bg-slate-200` / `text-slate-700`）

## 実装方針

### 1. 確認ダイアログの削除
- `isPhotoConfirmDialogOpen` の状態管理を削除
- `handlePhotoConfirmYes` / `handlePhotoConfirmNo` を削除
- ステータス選択後、直接ボトムシートを表示

### 2. ボトムシートの拡張
- `InspectionStatusBottomSheet` に写真撮影オプションを追加
- 「写真を撮影」「動画を録画」「スキップ」の選択肢を提供
- 写真撮影が必要なステータス（exchange, repair, adjust）の場合のみ表示

### 3. メディアUIの自動表示
- 写真撮影後、自動的にメディアUIを表示
- 追加ボタンは既存のメディアの横に表示

### 4. UIの一貫性確保
- すべてのボトムシートで同じデザインシステムを使用
- フォントサイズ、ボタンサイズ、アイコンサイズを統一

## 期待される効果

1. **操作ステップの削減**: 5ステップ → 3ステップ（40%削減）
2. **UIの一貫性向上**: すべてボトムシートで統一
3. **ユーザビリティの向上**: 直感的で分かりやすい操作フロー
4. **モバイル最適化**: ボトムシートはモバイルで使いやすい

## 実装優先度

1. **高**: 確認ダイアログの削除とボトムシートへの統合
2. **中**: メディアUIの自動表示
3. **低**: デザインの細かい調整
