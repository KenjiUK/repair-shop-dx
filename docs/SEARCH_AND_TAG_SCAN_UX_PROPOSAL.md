# 検索バー・タグスキャン UX改善提案（2025年ベストプラクティス）

## 現状分析

### 検索バーの用途
- 顧客名・車名・ナンバーでの検索（一覧から案件を探す）
- タグIDでの検索（手入力）

### タグスキャンの用途
- QRコードスキャン（カメラ起動）
- タグID手入力
- **案件に直接遷移**（検索とは異なる操作フロー）

### 問題点
1. **機能の重複**: 検索バーでタグID検索可能なのに、別途タグスキャンボタンが存在
2. **画面スペースの無駄**: 常時表示で画面を占有
3. **2025年トレンドとの不一致**: Progressive Disclosureの原則に反する

## 2025年最新UI/UXトレンド

### 1. **Progressive Disclosure（段階的開示）**
- 必要な時に必要な機能を表示
- 初期表示は最小限に

### 2. **Unified Search Experience**
- 検索とスキャンを1つのインターフェースに統合
- 検索バー内にスキャン機能を統合

### 3. **Contextual Actions**
- 操作の文脈に応じた機能配置
- 使用頻度に応じた表示

## 改善提案

### 提案A: 折りたたみ式検索バー + スキャン統合（推奨）

**実装:**
- 検索アイコンボタンで検索バーを展開/折りたたみ
- 検索バー内にスキャンアイコンを配置
- タグスキャンボタンを削除

**メリット:**
- 画面スペースの節約
- 機能の統合
- 2025年トレンドに合致

### 提案B: 検索バー削除 + フィルター強化

**実装:**
- 検索バーを削除
- ステータスフィルターと入庫区分フィルターで十分
- タグスキャンは別途ボタン（使用頻度が高い場合のみ）

**メリット:**
- 最もシンプル
- 画面スペースの最大化
- フィルターで十分な場合に有効

### 提案C: 検索バー統合（現在の改善版）

**実装:**
- 検索バー右側にスキャンアイコンを統合
- タグスキャンボタンを削除
- 検索バーは常時表示

**メリット:**
- 最小限の変更
- 機能の統合

## 最終推奨

**提案A（折りたたみ式検索バー + スキャン統合）を推奨**

理由:
1. 検索機能は必要だが、常時表示は不要
2. タグスキャンは検索バーに統合可能
3. 画面スペースを節約
4. 2025年のトレンドに合致
5. モバイルでの操作性向上
