# UIUX改善最終提案書

## 📋 概要

本ドキュメントは、UIUX包括的レビュー報告書に基づき、整備工場DXプラットフォームのUIUX改善に関する最終提案をまとめたものです。

**対象範囲:** 全ページ（12ファイル）、全コンポーネント（92ファイル以上）  
**レビュー実施日:** 2024年12月  
**提案日:** 2024年12月

**重要:** 本提案は**40歳以上のユーザー**（特に現場で作業する整備士・スタッフ）を主な対象としており、世界最高水準の自動車DXアプリのUIUXベストプラクティスに基づいています。

---

## 🎯 改善の目的

1. **一貫性の向上**: デザインシステムの統一による保守性とユーザビリティの向上
2. **情報の見やすさ**: レイアウト最適化による情報密度とスキャン可能性の向上
3. **操作性の向上**: ユーザー視点に立ったUI設計とアクセシビリティの向上（**40歳以上ユーザーに最適化**）
4. **現場DXアプリとしての最適化**: 業務フローに沿ったUI/UXの実現
5. **視認性と操作性の最適化**: 現場での使用（手袋着用、屋外環境）を考慮した設計

## 🎨 1. デザインシステムの統一

### 1.1 タイポグラフィスケール（40歳以上ユーザー最適化版）

#### 推奨システム

```typescript
// design-system.ts
// ⚠️ 40歳以上ユーザー向け: 最小フォントサイズ16px（text-base）以上を推奨
export const typography = {
  h1: "text-2xl font-bold text-slate-900",           // ページタイトル（24px）
  h2: "text-xl font-bold text-slate-900",            // セクションタイトル（20px）
  h3: "text-lg font-semibold text-slate-900",        // カードタイトル（18px）
  body: "text-base text-slate-800 font-normal",      // 本文（16px、コントラスト向上: slate-700→slate-800）
  label: "text-base font-medium text-slate-800",     // ラベル（16px、text-sm→text-baseに変更）
  caption: "text-base text-slate-700",               // 補足情報（16px、text-sm→text-baseに変更）
  // ❌ text-xs (12px) は使用禁止（視認性が低すぎる）
  // ❌ text-sm (14px) は最小限の使用（ラベルなど重要な情報には使用しない）
};
```

#### 40歳以上ユーザー向けの考慮事項

1. **最小フォントサイズ: 16px (text-base)**
   - WCAG 2.1推奨: 本文は最小16px
   - 40歳以上ユーザー: 視力低下を考慮し、16px以上を推奨
   - `text-xs` (12px), `text-sm` (14px) の使用を最小限に

2. **コントラスト比の向上**
   - `text-slate-500` (低コントラスト) → 使用禁止
   - `text-slate-600` (中コントラスト) → `text-slate-700` または `text-slate-800` に変更
   - WCAG AAレベル: 4.5:1以上を確保

3. **フォントウェイトの最適化**
   - 重要な情報は `font-semibold` または `font-bold` を使用
   - 視認性を向上させるため、適度に太字を使用

#### アクションプラン

- ✅ すべてのページタイトルを `text-xl` → `text-2xl` に統一
- ✅ カードタイトルを `text-lg font-semibold` に統一（`text-base` を排除）
- ✅ セクションタイトルを `text-xl font-bold` に統一
- ✅ **ラベルを `text-sm` → `text-base` に変更**（40歳以上ユーザー向け）
- ✅ **補足情報を `text-sm` → `text-base` に変更**（40歳以上ユーザー向け）
- ✅ **`text-xs` の使用を禁止**（視認性が低すぎる）
- ✅ テキストカラーを適切に使い分け（slate-900 → slate-800 → slate-700、slate-600以下は使用しない）

**影響範囲**: 約150箇所  
**優先度**: 高（40歳以上ユーザーにとって最重要）

### 1.2 カラーパレットの統一（40歳以上ユーザー最適化版）

#### 推奨システム

```typescript
// プライマリアクション（統一）
variant="default" // bg-primary を使用（カスタム bg-blue-600 を排除）

// ステータスカラー（統一、コントラスト比を高く確保）
// 警告: amber のみを使用（yellow を排除）
className="bg-amber-50 text-amber-900 border-amber-300" // text-amber-800→text-amber-900に変更（コントラスト向上）

// 情報
className="bg-blue-50 text-blue-900 border-blue-300" // text-blue-800→text-blue-900に変更（コントラスト向上）

// 成功
className="bg-green-50 text-green-900 border-green-300" // text-green-800→text-green-900に変更（コントラスト向上）

// エラー
className="bg-red-50 text-red-900 border-red-300" // text-red-800→text-red-900に変更（コントラスト向上）
```

#### 40歳以上ユーザー向けの考慮事項

1. **コントラスト比の向上**
   - WCAG AAレベル: 4.5:1以上を確保
   - テキストカラーを `-800` → `-900` に変更（コントラスト向上）
   - ボーダーカラーを `-200` → `-300` に変更（視認性向上）

2. **色の識別性**
   - 色だけでなく、アイコンやテキストで情報を表現（色覚異常への対応）
   - 重要な情報は複数の手がかり（色、アイコン、テキスト）で表現

#### アクションプラン

- ✅ すべての `bg-blue-600` を `variant="default"` (bg-primary) に置き換え
- ✅ すべての `yellow` カラーを `amber` に統一
- ✅ **ステータスカラーのテキストを `-800` → `-900` に統一**（40歳以上ユーザー向け、コントラスト向上）
- ✅ **ボーダーカラーを `-200` → `-300` に統一**（視認性向上）
- ✅ **`text-slate-500`, `text-slate-600` の使用を最小限に**（コントラストが低すぎる）

**影響範囲**: 約50箇所  
**優先度**: 高（40歳以上ユーザーにとって最重要）

### 1.3 ボタンサイズシステムの統一（40歳以上ユーザー最適化版）

#### 推奨システム

```typescript
// ⚠️ 40歳以上ユーザー向け: 最小タッチターゲットサイズ48px（h-12）以上を推奨
// WCAG 2.5.5 Target Size (Enhanced): 44x44px（最小）、48x48px（推奨）

// 標準ボタン（推奨）
size="default" // h-12 (48px) - 40歳以上ユーザー向けに拡大

// コンパクトボタン（デスクトップのみ、最小限の使用）
size="sm" // h-10 (40px) - デスクトップのみ、補助的なボタンに限定

// 大ボタン（重要アクション用）
size="lg" // h-14 (56px) - 最重要アクション用（推奨サイズ）

// 超大型ボタン（現場作業向け）
size="xl" // h-16 (64px) - 手袋着用時や緊急時の操作向け

// ❌ h-8 (32px), h-9 (36px) は使用禁止（タッチターゲットが小さすぎる）
```

#### 40歳以上ユーザー向けの考慮事項

1. **最小タッチターゲットサイズ: 48px (h-12)**
   - WCAG 2.5.5 Target Size (Enhanced): 48x48px推奨
   - 40歳以上ユーザー: 指先の感覚低下を考慮
   - **現場作業**: 手袋着用時も操作可能なサイズを確保

2. **ボタン間隔の確保**
   - ボタン間の間隔を8px以上確保（誤操作防止）
   - 重要なボタンは周囲に十分なスペースを確保

3. **ボタンテキストサイズ**
   - ボタン内テキストは最小16px (text-base) 以上
   - `text-sm` (14px) の使用を避ける

#### アクションプラン

- ✅ **標準ボタンを `h-9` (36px) → `h-12` (48px) に変更**（40歳以上ユーザー向け）
- ✅ **重要アクションは `h-14` (56px) または `h-16` (64px) を使用**（現場作業向け）
- ✅ **`h-8` (32px), `h-9` (36px) の使用を禁止**（タッチターゲットが小さすぎる）
- ✅ モバイルでは最小タッチターゲットサイズ（48px = h-12）を必須に
- ✅ ボタン内テキストサイズを `text-base` (16px) 以上に統一

**影響範囲**: 約30箇所（ボタンサイズ変更により影響範囲が拡大）  
**優先度**: 高（40歳以上ユーザーにとって最重要）

### 1.4 アイコンサイズの統一（40歳以上ユーザー最適化版）

#### 推奨システム

```typescript
// ⚠️ 40歳以上ユーザー向け: アイコンサイズを拡大
export const iconSizes = {
  button: "size-5",     // ボタン内アイコン（size-4→size-5に拡大、20px）
  title: "size-6",      // カードタイトル・セクションタイトルアイコン（size-5→size-6に拡大、24px）
  large: "size-7",      // 大きなアイコン・アラート（size-6→size-7に拡大、28px）
  badge: "size-4",      // バッジ内アイコン（size-3→size-4に拡大、16px）
  // ❌ size-3 (12px) は使用禁止（視認性が低すぎる）
};
```

#### 40歳以上ユーザー向けの考慮事項

1. **アイコンサイズの拡大**
   - 最小アイコンサイズ: 16px (size-4)
   - `size-3` (12px) の使用を禁止
   - 重要なアイコンは20px (size-5) 以上

2. **アイコンとテキストのバランス**
   - アイコンサイズはテキストサイズの1.2-1.5倍を推奨
   - ボタン内では、アイコンとテキストの高さを揃える

#### アクションプラン

- ✅ すべてのアイコンサイズを `size-*` 記法に統一（または `h-* w-*` に統一）
- ✅ **カードタイトルアイコンを `size-5` → `size-6` に変更**（40歳以上ユーザー向け）
- ✅ **ボタン内アイコンを `size-4` → `size-5` に変更**（40歳以上ユーザー向け）
- ✅ **バッジ内アイコンを `size-3` → `size-4` に変更**（40歳以上ユーザー向け）
- ✅ **`size-3` (12px) の使用を禁止**（視認性が低すぎる）

**影響範囲**: 約150箇所  
**優先度**: 中（40歳以上ユーザーにとって重要）
