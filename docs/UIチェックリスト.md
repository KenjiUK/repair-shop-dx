# UIチェックリスト

このドキュメントは、実装されたUIが設計仕様通りかどうかを確認するためのチェックリストです。

## 📋 チェック方法

1. 各画面を開いて、このチェックリストの項目を確認
2. 仕様通りかどうかを✅（問題なし）または❌（問題あり）で記録
3. 問題がある場合は、スクリーンショットを撮って詳細を記録

---

## 🎨 共通UI要素（全画面共通）

### デザインシステム
- [ ] **カラーパレット**: 指定された色が使用されているか
  - プライマリ: `slate-900`, `slate-600`, `slate-500`, `slate-300`
  - セマンティック: `green-600`（成功）, `amber-600`（警告）, `red-600`（エラー）, `blue-600`（情報）
  - 背景: `white`, `slate-50`, `slate-100`
  - 参照: [UI_UX_GUIDELINES.md](./UI_UX_GUIDELINES.md#1-1-カラーパレット)

- [ ] **タイポグラフィ**: フォントサイズとウェイトが指定通りか
  - `text-2xl font-bold`: ページタイトル
  - `text-xl font-bold`: セクションタイトル
  - `text-base`: 本文
  - `text-sm`: 補助テキスト
  - 参照: [UI_UX_GUIDELINES.md](./UI_UX_GUIDELINES.md#1-2-タイポグラフィ)

- [ ] **アイコンサイズ**: 指定されたサイズが使用されているか
  - `h-5 w-5` (20px): セクションタイトルアイコン、第1階層
  - `h-4 w-4` (16px): 第2階層、ボタン内アイコン
  - `h-3.5 w-3.5` (14px): バッジ内アイコン
  - 参照: [UI_UX_GUIDELINES.md](./UI_UX_GUIDELINES.md#9-2-アイコンサイズ)

- [ ] **スペーシング**: 適切な間隔が設定されているか
  - `space-y-6`: セクション間
  - `space-y-4`: 項目間
  - `gap-2`, `gap-4`: フレックスアイテム間
  - 参照: [UI_UX_GUIDELINES.md](./UI_UX_GUIDELINES.md#1-3-スペーシング)

- [ ] **ボーダーとシャドウ**: 統一感があるか
  - `border border-slate-300`: 標準ボーダー
  - `rounded-lg`, `rounded-xl`: 角丸
  - `shadow`: カードのシャドウ
  - 参照: [UI_UX_GUIDELINES.md](./UI_UX_GUIDELINES.md#1-4-ボーダーとシャドウ)

### ヘッダー（AppHeader）
- [ ] **ロゴとブランド名**: 左上に表示されているか
- [ ] **スクロール時の動作**: スクロール時に適切に縮小・展開するか
  - 下スクロール: 15px累積で縮小
  - 上スクロール: 30px累積で展開
- [ ] **戻るボタン**: トップページ以外で表示されているか
- [ ] **ステータスバッジ**: 適切な色とテキストで表示されているか
  - 参照: [STATUS_COLOR_RULES.md](./STATUS_COLOR_RULES.md)

### ボタン
- [ ] **プライマリボタン**: `bg-slate-900 text-white hover:bg-slate-800`
- [ ] **セカンダリボタン**: `variant="outline" border-slate-300`
- [ ] **ボタンサイズ**: `h-10` または `h-12`（アクションボタン）
- [ ] **ローディング状態**: スピナーアイコンが表示されるか

### カード
- [ ] **基本カード**: `border border-slate-300 rounded-xl shadow`
- [ ] **情報カード**: 背景色とボーダーが適切か

### バッジ
- [ ] **ステータスバッジ**: 色とテキストが適切か
  - 参照: [STATUS_COLOR_RULES.md](./STATUS_COLOR_RULES.md)
- [ ] **入庫区分バッジ**: カテゴリー別の色が適切か
  - 参照: [SERVICE_KIND_COLOR_RULES.md](./SERVICE_KIND_COLOR_RULES.md)

## 📱 画面別チェックリスト

### Phase 0: 事前チェックイン画面 (`/customer/pre-checkin/[id]`)

#### レイアウト
- [ ] ページタイトル: "事前チェックイン" が表示されているか
- [ ] 顧客名が表示されているか（例: "田中様のご予約について"）

#### 車両選択セクション
- [ ] 「今回ご入庫のお車」セクションタイトル（Carアイコン付き）
- [ ] 既存車両リストが表示されているか
- [ ] ラジオボタンで選択できるか
- [ ] 「別の車（新規）」ボタンがあるか
- [ ] 新規車両入力時: 車種名入力フィールド、車検証写真アップロード

#### 顧客情報確認セクション
- [ ] 「お客様情報の確認」セクションタイトル（CheckCircle2アイコン付き）
- [ ] 住所入力フィールド（初期値: Zohoから取得）
- [ ] 電話番号入力フィールド（初期値: Zohoから取得）
- [ ] 変更時に「変更あり」バッジが表示されるか

#### メール配信セクション
- [ ] 「メール配信について」セクションタイトル（CheckCircle2アイコン付き）
- [ ] チェックボックス: "メールでのお知らせを受け取ることに同意します"

#### 問診セクション
- [ ] 「問診」セクションタイトル（CheckCircle2アイコン付き）
- [ ] 走行距離入力フィールド（数値型、km単位の説明あり）
- [ ] 不具合・気になるところ（Textarea、プレースホルダーあり）
- [ ] 代車希望チェックボックス

#### 送信ボタン
- [ ] 「送信」ボタンが表示されているか
- [ ] 送信中はスピナーアイコンと「保存中...」が表示されるか

## 🎯 レスポンシブデザインチェック

### モバイル（〜640px）
- [ ] ヘッダーが適切に折りたたまれるか
- [ ] カードが1列で表示されるか
- [ ] ボタンがタッチしやすいサイズか（最小44px×44px）
- [ ] テキストが読みやすいサイズか

### タブレット（640px〜1024px）
- [ ] 2列レイアウトが適切に表示されるか
- [ ] グリッドが適切に調整されるか

### デスクトップ（1024px〜）
- [ ] 最大幅が設定されているか（`max-w-4xl`、`max-w-7xl`など）
- [ ] コンテンツが中央配置されているか

- [ ] **フォーカス表示**: フォーカスインジケーターが明確か

## 📝 チェック結果の記録

各チェック項目について、以下を記録してください：

- ✅ **問題なし**: 仕様通りに実装されている
- ❌ **問題あり**: 仕様と異なる、またはバグがある
- ⚠️ **要改善**: 仕様通りだが、UX改善の余地がある

### 問題記録フォーマット

```
画面名: [画面名]
セクション: [セクション名]
問題: [問題の詳細説明]
スクリーンショット: [ファイルパス]
優先度: [高/中/低]
```
